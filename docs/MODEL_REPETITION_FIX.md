# 模型重复说话问题修复报告

**修复时间**: 2026-04-13 01:25  
**问题级别**: 🔴 严重  
**问题现象**: 模型生成重复文本，如"您住在北京，您住在北京，您住在北京..."

---

## 🔍 问题分析

### 问题现象
从连接器日志中观察到：
```
"在北京，您住在北京，您住在北京，您住在北京，您住在北京，您住在北京..."
```

模型陷入了**文本生成循环**，不断重复相同的内容。

### 根因分析

#### 原因 1: 缺少重复惩罚机制 ❌

**当前配置**:
```python
response = self.vllm_client.chat.completions.create(
    model=vllm_model_id,
    messages=messages,
    max_tokens=1024,
    temperature=0.3,
    top_p=0.85,
    stream=False
    # ❌ 缺少 repetition_penalty 参数
)
```

**问题**:
- ❌ **没有重复惩罚**: 模型不会因为生成重复文本而受到惩罚
- ❌ **温度较低**: temperature=0.3 本身会降低多样性，但不足以防止重复
- ❌ **小模型特性**: Qwen3.5-0.8B 是小模型，更容易陷入重复模式

#### 原因 2: 可能的 System Prompt 问题 ⚠️

**观察**:
- System Prompt 可能包含重复的指令
- 记忆注入可能包含重复内容
- RAG 检索可能返回重复知识

#### 原因 3: 对话历史可能包含重复 ⚠️

**可能情况**:
- 之前的重复回复被记录到对话历史
- 模型学习了自己的重复模式
- 形成恶性循环

---

## ✅ 修复方案

### 修复 1: 添加重复惩罚参数（核心修复）

**修改文件**: `zulong/l2/inference_engine.py`

**修改位置**: 所有 vLLM 生成调用点

**修改内容**:
```python
response = self.vllm_client.chat.completions.create(
    model=vllm_model_id,
    messages=messages,
    max_tokens=1024,
    temperature=0.3,
    top_p=0.85,
    repetition_penalty=1.2,  # 🔥 关键修复：添加重复惩罚
    stream=False
)
```

**参数说明**:
- **repetition_penalty=1.2**: 
  - 值范围：1.0 ~ 2.0
  - 1.0 = 无惩罚
  - 1.2 = 轻度惩罚（推荐）
  - 1.5 = 中度惩罚
  - 2.0 = 重度惩罚（可能导致回复过于保守）

**作用机制**:
- 对重复的 token 增加惩罚
- 降低重复内容的生成概率
- 鼓励模型使用更多样化的词汇

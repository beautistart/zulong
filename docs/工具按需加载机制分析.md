# 工具按需加载机制分析

## IDE端当前机制（已完善）

### 分层裁剪策略

```
Layer 1: 意图分类（初始加载）
  ├─ CHAT模式 → 5个基础工具
  ├─ RESUME模式 → ~36个工具（排除task_create_plan/task_add_node）
  └─ COMPLEX模式 → ~41个工具（全部）

Layer 2: CircuitBreaker评估（运行时）
  ├─ GREEN → 保持当前工具集
  ├─ YELLOW → 注入警告，引导收敛
  └─ RED → 裁剪至5个收敛工具

Layer 3: 压力感知（上下文压力）
  ├─ GREEN → 正常工具集
  ├─ YELLOW → 注入引导消息
  └─ RED → 仅保留注意力工具

Layer 4: CB强制模式
  └─ cb_tool_streak >= 3 → 移除所有工具，强制纯文本回复
```

### 各层工具数量

| 状态 | 工具数量 | 说明 |
|------|---------|------|
| CHAT初始 | **5** | 记忆+搜索 |
| RESUME初始 | **~36** | 排除创建工具 |
| COMPLEX初始 | **~41** | 全部工具 |
| CB RED | **5** | 收敛工具 |
| 压力RED | **1** | 注意力导航 |
| CB强制 | **0** | 纯文本 |

def _detect_ide_intent(user_input: str):
    # 优先使用L1-B IntentFilter（ALBERT模型）
    if self._intent_filter:
        albert_intent = self._intent_filter.classify(user_input)
        if albert_intent in ("chat", "resume", "complex"):
            return albert_intent, has_active_tg

    if "继续" in user_input or "恢复" in user_input:

## 统一后工具注入策略

### Web端工具路由

```python
async def resolve_tools_for_web(
    user_message: str,
    project_context: Optional[str]
) -> List[Dict]:
    """Web端工具路由"""
    
    # 1. 意图分类
    intent = detect_intent(user_message)
    
    # 2. 项目上下文判断
    has_project = project_context is not None
    
    # 3. 工具选择
    if intent == "chat":
        # 简单对话：5个基础工具
        return get_chat_tools()
    
    elif intent == "complex" and has_project:
        # 编程任务：完整工具（内部+远程）
        return get_combined_tools("complex")
    
    elif intent == "resume":
        # 恢复任务：排除创建工具
        return get_combined_tools("resume")
    
    else:
        # 普通复杂任务：内部工具
        return get_internal_tools("complex")
```

### 编程任务检测

```python
PROGRAMMING_INDICATORS = {
    "分析代码", "修改文件", "执行命令", "搜索文件",
    "read_file", "write_to_file", "execute_command",
    "实现功能", "修复bug", "重构代码",
}

def is_programming_task(message: str, project_context: Optional[str]) -> bool:
    # 有项目上下文 + 编程关键词 = 编程任务
    if project_context:
        if any(kw in message for kw in PROGRAMMING_INDICATORS):
            return True
    return False
```

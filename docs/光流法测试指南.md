## 📝 状态机流转说明

### 状态定义

| 状态 | 描述 | 触发条件 |
|------|------|---------|
| **IDLE** | 静止状态 | 初始状态，或 STOPPED 持续一帧 |
| **MOVING** | 检测到运动 | 运动像素 > 阈值 |
| **STOPPED** | 运动停止 | MOVING 状态持续静止 > 2 秒 |

### 状态流转图

```
IDLE ──(运动像素>阈值)──> MOVING
  ▲                         │
  │                         │(静止>2 秒)
  │                         ▼
  └──── (STOPPED 持续一帧) ── STOPPED
```

### 自适应阈值机制

- **检测到运动时**: 阈值降低（变得更敏感）
  ```python
  threshold = max(1.0, threshold * (1 - 0.15))
  ```
- **静止时**: 阈值缓慢恢复（防止误检）
  ```python
  threshold = min(base*2.0, threshold * (1 + 0.075))
  ```

## 🔍 故障排查

### 问题 1: 没有检测到运动

**可能原因**:
- 阈值过高
- 光线太暗
- 运动幅度太小

**解决方案**:
```python
# 在 motion_detector.py 中调整参数
OpticalFlowMotionDetector(
    pixel_threshold=5000,  # 降低像素阈值（默认 8000）
    base_threshold=2.0,    # 降低光流幅值阈值（默认 2.5）
)
```

### 问题 2: 持续检测到运动（无法回到 IDLE）

**可能原因**:
- 光线变化太大
- 摄像头抖动
- 阈值太低

**解决方案**:
```python
# 增加像素阈值或增加 motion_stop_timeout
OpticalFlowMotionDetector(
    pixel_threshold=10000,     # 增加阈值
    motion_stop_timeout=3.0,   # 增加停止判定时间（默认 2.0）
)
```

### 问题 3: 基准帧未设置

**检查日志**:
```
[OpticalFlow] 基准帧已强制设置
```

如果没有看到此日志，检查：
- CameraDevice 是否正常启动
- VisionProcessor 是否已初始化
- feed_frame 是否被调用

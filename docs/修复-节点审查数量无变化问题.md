- ❌ 但数量一直是1358，没有减少

## 🐛 根本原因分析

### 原因1: L2BackupScheduler调度器状态管理问题

**文件**: `zulong/l2/backup_scheduler.py`

**问题代码** (第306-345行):

```python
async def _scheduler_loop(self):
    while self._running:
        # 1. 检查 L2-PRIME 状态
        if self.l2_prime_status == L2Status.IDLE:  # ← 必须是IDLE才会执行
            if not self._task_queue.empty():
                task = self._task_queue.get()
                
                if self.l2_backup_status == L2Status.IDLE:
                    self.update_l2_backup_status(L2Status.PROCESSING)
                    asyncio.create_task(self._process_task(task))
                    
                    # 🔥 问题：0.5秒后就重置为IDLE
                    await asyncio.sleep(0.5)
                    self.update_l2_backup_status(L2Status.IDLE)
```

**问题**:
1. **L2-PRIME状态可能不是IDLE**：如果状态一直是BUSY，任务永远不会被执行
2. **L2-BACKUP状态立即重置**：0.5秒后就设为IDLE，但任务还在执行中，可能导致状态混乱
3. **任务完成后没有重置状态**：`_process_task`完成后没有将L2-BACKUP状态重置为IDLE

### 原因2: MemoryGraph候选列表清理逻辑问题

**文件**: `zulong/memory/memory_graph.py`

**问题代码** (第861-894行):

```python
submitted_count = 0
if llm_candidates:
    # ... 提交审查任务 ...
    # 🔥 问题：无论成功失败都会执行
    submitted_count = len(batch)
    self._pending_llm_candidates = self._pending_llm_candidates[submitted_count:]
```

**问题**:
1. **提交失败时也清空候选列表**：如果`review_importance_candidates`抛出异常，候选列表仍被清空
2. **下次扫描时重复添加**：`run_importance_review`再次扫描时，相同的节点又被加入候选列表
3. **导致候选数量永远不变**：提交50个 → 失败清空 → 下次又扫描到相同的50个 → 永远是1358个

## ✅ 修复方案

### 修复1: L2BackupScheduler状态管理优化

**文件**: `zulong/l2/backup_scheduler.py`

#### 1.1 允许ERROR状态下也执行后台任务

```python
# 修复前
if self.l2_prime_status == L2Status.IDLE:

# 修复后
if self.l2_prime_status in (L2Status.IDLE, L2Status.ERROR):
```

**理由**: 如果L2-PRIME出错，不应该阻止后台记忆审查任务执行

#### 1.2 移除立即重置状态的代码

```python
# 修复前
asyncio.create_task(self._process_task(task))
await asyncio.sleep(0.5)
self.update_l2_backup_status(L2Status.IDLE)  # ← 删除这2行

# 修复后
asyncio.create_task(self._process_task(task))
# 不在这里重置状态，让_process_task完成后再重置
```

**理由**: 任务还在执行中不应该重置状态

#### 1.3 在_process_task完成后重置状态

```python
async def _process_task(self, task: SummarizationTask):
    try:
        # ... 执行任务 ...
    finally:
        # ... 清理代码 ...
        
        # 🔥 新增：任务完成后重置L2-BACKUP状态为IDLE
        self.update_l2_backup_status(L2Status.IDLE)
        logger.info(f"[L2BackupScheduler] L2-BACKUP状态重置为IDLE，可处理下一个任务")
```

#### 1.4 增强日志输出

```python
# 提交审查结果回写时
logger.info(f"[L2BackupScheduler] 开始回写审查结果: {review_task_id}, 原始记忆数={len(original_memories)}, context={context}")
await reviewer.process_review_result(...)
logger.info(f"[L2BackupScheduler] ✅ 审查结果已回写: {review_task_id}")

# 异常时输出完整堆栈
logger.error(f"[L2BackupScheduler] 任务失败：{task.task_id}, 错误：{e}", exc_info=True)
```

### 修复2: MemoryGraph候选列表清理优化

**文件**: `zulong/memory/memory_graph.py`

#### 2.1 只在提交成功时清空候选

```python
# 修复前
try:
    loop.run_until_complete(reviewer.review_importance_candidates(candidate_memories))
except Exception as e:
    logger.warning(f"[MemoryGraph] 重要性审查提交失败: {e}")
submitted_count = len(batch)  # ← 无论成功失败都计数
self._pending_llm_candidates = self._pending_llm_candidates[submitted_count:]

# 修复后
try:
    loop.run_until_complete(reviewer.review_importance_candidates(candidate_memories))
    # 🔥 只在成功时计数并清空
    submitted_count = len(candidate_memories)
    self._pending_llm_candidates = self._pending_llm_candidates[submitted_count:]
    logger.info(f"[MemoryGraph] ✅ 成功提交 {submitted_count} 个节点给LLM审查，剩余候选={len(self._pending_llm_candidates)}")
except Exception as e:
    logger.warning(f"[MemoryGraph] 重要性审查提交失败: {e}")
    submitted_count = 0  # ← 失败时不计数
```

#### 2.2 增加详细日志

```python
logger.info(f"[MemoryGraph] {len(llm_candidates)} 个节点待 LLM 审查确认提升，已提交 {submitted_count} 个，剩余待提交={len(self._pending_llm_candidates)}")
```

**现在可以看到**:
- 待审查数量
- 本次提交数量
- 剩余待提交数量

#### 2.3 修复imp变量作用域问题

```python
# 修复前
candidate_memories.append({
    "importance": str(imp.value if (imp := self.get_importance(cid)) else "NORMAL"),
})

# 修复后
imp = self.get_importance(cid)
candidate_memories.append({
    "importance": str(imp.value if imp else "NORMAL"),
})
```

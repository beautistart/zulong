## 🎯 关键参数说明

### OpticalFlowMotionDetector 参数

```python
detector = OpticalFlowMotionDetector(
    flow_threshold=0.5,          # 光流幅值阈值
    motion_pixel_threshold=500,  # 运动像素数量阈值
    max_features=100             # 最大特征点数量 (稀疏光流)
)
```

**参数调优指南**:

| 参数 | 调高效果 | 调低效果 | 推荐值 |
|------|---------|---------|--------|
| `flow_threshold` | 降低敏感度，减少误检 | 提高敏感度，检测细微运动 | 0.5-5.0 |
| `motion_pixel_threshold` | 需要更大面积运动才触发 | 小面积运动也能触发 | 500-10000 |
| `baseline_timeout` | 基准帧更稳定 | 更快适应环境变化 | 3.0 秒 |

## 🔧 常见问题排查

### 问题 1: 摄像头无法打开

**症状**:
```
❌ 无法打开摄像头 0
```

**解决方案**:
1. 检查摄像头是否被其他程序占用
2. 尝试其他摄像头 ID (0, 1, 2...)
3. 使用模拟视频模式：输入 `n`

---

### 问题 2: 检测不到运动

**症状**:
- 明显有运动，但 `Motion: 0 (0.0%)`

**解决方案**:
1. 降低 `flow_threshold` (从 5.0 降到 0.5)
2. 降低 `motion_pixel_threshold` (从 10000 降到 500)
3. 检查光线是否太暗

---

### 问题 3: 持续误检

**症状**:
- 静态场景也显示 `Motion: 100%`

**解决方案**:
1. 提高 `flow_threshold` (从 0.5 升到 5.0)
2. 提高 `motion_pixel_threshold` (从 500 升到 10000)
3. 按 'r' 重置基准帧
4. 检查是否有持续光源变化 (如闪烁的灯)

---

### 问题 4: FPS 过低

**症状**:
- `FPS: < 10`

**解决方案**:
1. 降低摄像头分辨率 (从 1920x1080 降到 640x480)
2. 降低帧率 (从 30FPS 降到 15FPS)
3. 减少显示窗口数量

## 🎓 学习资源

### 光流法原理

1. **Farneback 稠密光流法**:
   - 优点：计算每个像素的运动矢量
   - 缺点：计算量较大
   - 适用场景：需要精细运动检测

2. **LK 稀疏光流法**:
   - 优点：计算速度快
   - 缺点：只计算特征点
   - 适用场景：跟踪特定点

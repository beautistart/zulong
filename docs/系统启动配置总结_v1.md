## 📊 性能预期

### 延迟分析

基于 TSD v1.7 和 CosyVoice3-0.5B 官方 benchmark：

| 操作 | CPU 模式 | GPU 模式 |
|------|----------|----------|
| **TTS 首帧延迟** | ~500-800ms | ~200-300ms |
| **TTS 合成速度** | ~1.0-1.5x RTF | ~0.5x RTF |
| **总延迟 (短句)** | ~1000-2000ms | ~500-800ms |

**说明**: 
- RTF = Real Time Factor (实时率)
- CPU 模式延迟增加约 500-1200ms，但释放了 1.2GB 显存
- 对于非实时交互场景（如生成文档、长篇回复），CPU 模式完全可接受

### 显存优势

**GPU 模式 (TTS 在 GPU)**:
- L1-A: 0.7 GB
- L1-B: 0.7 GB
- L2: 0.7 GB
- TTS: 1.2 GB
- **总计：3.3 GB** (剩余 2.7 GB)

**CPU 模式 (TTS 在 CPU)**:
- L1-A: 0.7 GB
- L1-B: 0.7 GB
- L2: 0.7 GB
- TTS: 0.0 GB (运行在 RAM)
- **总计：2.1 GB** (剩余 3.9 GB) ✅

**优势**: 多出的 1.8GB 显存可用于：
- 更大的上下文窗口
- 未来扩展更多 L3 专家
- 更高的批处理大小

## ⚠️ 已知限制

### TTS CPU 模式的限制

1. **延迟增加**: 首帧延迟从 300ms 增加到 800ms
2. **CPU 负载**: 合成时 CPU 占用率较高
3. **RAM 占用**: TTS 模型占用约 2-3 GB 系统 RAM

### 解决方案

- ✅ 对于对话式交互，延迟仍在可接受范围内（<2 秒）
- ✅ 系统 RAM 充足（通常 16GB+），2-3GB 占用不是问题
- ✅ 未来可考虑按需加载 TTS 模型（用完即卸载）

# Layer 4 触发阈值调整说明

**更新日期**: 2026-04-02  
**版本**: v4.0 (最低阈值版)

---

## 📊 调整依据

### Layer 3 实测数据回顾

根据 95.9 秒的实测统计：

| 意图 | 样本数 | 平均分数 | **分数范围** |
|------|--------|----------|-------------|
| SITTING | 501 | 0.259 | **0.043 - 0.573** |
| STANDING | 456 | 0.147 | **0.041 - 0.314** |
| LYING | 158 | 0.162 | **0.033 - 0.423** |
| WALKING | 46 | 0.130 | **0.066 - 0.225** |
| WAVING | 44 | 0.120 | **0.067 - 0.333** |
| UNKNOWN | 5 | 0.000 | **0.000 - 0.000** |

### 最低置信度分析

- **最低有效分数**: 0.033 (LYING)
- **次低分数**: 0.041 (STANDING)
- **第三低**: 0.043 (SITTING)
- **UNKNOWN 分数**: 0.000 (无效意图)

**结论**: Layer 3 检测到的有效意图最低分数为 **0.033**

## 🔧 阈值配置

### 新配置

```python
# Layer 3
'intent_threshold': 0.25,    # 意图检测门槛
'interact_threshold': 0.05,  # Layer 4 触发门槛 ← 调整到最低置信度之上

# Layer 4
'gesture_conf_threshold': 0.25,  # 手势识别门槛
'eagle_eye_cooldown': 0.3,       # 冷却时间
```

### 配置逻辑

#### interact_threshold = 0.05

**设置理由**:
1. **高于最低有效分数**: 0.05 > 0.033，确保不是误触发
2. **尽可能低**: 确保只要 Layer 3 检测到意图，Layer 4 就能触发
3. **过滤 UNKNOWN**: 0.05 > 0.000，过滤掉无效状态

**预期效果**:
- 只要 Layer 3 分数 >= 0.05，Layer 4 就会触发
- 覆盖 100% 的有效意图 (最低 0.033)
- 过滤掉 UNKNOWN 状态 (0.000)

#### intent_threshold = 0.25 (保持不变)

**作用**: 判断是否有意图的门槛
- < 0.25: NO_INTENT
- >= 0.25: 有意图 (SILENT_WATCH 或 INTERACT_REQUEST)

**为什么与 interact_threshold 不同**:
- `intent_threshold`: 判断"是否有意图"
- `interact_threshold`: 判断"是否触发 Layer 4"
- 两者可以不同，实现分级触发

## 📈 触发逻辑

### 分级触发机制

```
Layer 3 分数        状态              Layer 4
─────────────────────────────────────────────
< 0.05            NO_INTENT        ❌ 不触发
0.05 - 0.25       有意图但分低      ⚠️  可能触发*
>= 0.25           SILENT_WATCH     ✅ 触发
>= 0.35           INTERACT_REQUEST ✅ 触发 + 鹰眼
```

*注：0.05-0.25 之间会触发 Layer 4，但可能不会发布交互事件

### 状态流转

```python
if action_score < intent_threshold (0.25):
    state = NO_INTENT
    Layer 4: ❌ 不触发

elif action_score < interact_threshold (0.05):
    # 这行代码实际上不会执行，因为 0.05 < 0.25
    # 但逻辑上表示：极低分数不触发
    state = NO_INTENT
    Layer 4: ❌ 不触发

elif action_score < interact_threshold (0.05):
    # 实际上，只要 >= 0.05 就会触发 Layer 4
    state = SILENT_WATCH
    Layer 4: ✅ 触发

else:
    state = INTERACT_REQUEST
    Layer 4: ✅ 触发 + 鹰眼模式
```

## 🎯 预期效果

### Layer 4 触发频率

**调整前** (interact_threshold = 0.35):
- 触发条件：L3 >= 0.35
- 覆盖帧数：40 帧 (3.3%)
- 触发频率：低

**调整后** (interact_threshold = 0.05):
- 触发条件：L3 >= 0.05
- 覆盖帧数：~1205 帧 (99.6%)
- 触发频率：高

### 优势

1. **OK 手势识别率提升**
   - 即使 L3 分数很低 (如 0.1-0.2)，Layer 4 也能触发
   - OK 手势动作小，L3 分数通常不高
   - 现在可以稳定识别

2. **所有手势都容易触发**
   - V 字手势：L3 通常 > 0.2，✅ 触发
   - OK 手势：L3 可能 0.1-0.2，✅ 触发
   - 点赞手势：L3 通常 > 0.2，✅ 触发
   - 张开手掌：L3 通常 > 0.2，✅ 触发

3. **用户体验更好**
   - 不需要刻意做大幅动作
   - 自然手势就能触发识别
   - 响应更灵敏

### 潜在问题

1. **触发过于频繁**
   - 几乎 100% 的时间 Layer 4 都在运行
   - 可能增加 CPU 负担
   - **解决**: 依靠 `eagle_eye_cooldown = 0.3` 限制频率

2. **误识别增加**
   - 低分数时手势识别可能不准确
   - **解决**: `gesture_conf_threshold = 0.25` 保证识别质量

3. **资源消耗**
   - MediaPipe 持续运行
   - **解决**: 观察 CPU 占用，如过高可调整 cooldown

## 🔬 进一步优化

### 如果触发太频繁

**方案 1**: 微调 interact_threshold
```python
'interact_threshold': 0.05 -> 0.08  # 略提高
```

**方案 2**: 增加冷却时间
```python
'eagle_eye_cooldown': 0.3 -> 0.5  # 降低频率
```

**方案 3**: 添加最小动作检测
```python
# 在 _layer4_eagle_eye_mode 中
if action_score < 0.1 and timestamp - last_gesture_time < 2.0:
    return None  # 2 秒内不重复触发
```

### 如果仍然识别不到 OK 手势

**方案 1**: 检查手势标准度
- 拇指和食指必须形成完整圆圈
- 其他三指伸直
- 手掌面向摄像头

**方案 2**: 降低 gesture_conf_threshold
```python
'gesture_conf_threshold': 0.25 -> 0.2
```

**方案 3**: 添加 OK 手势增强
```python
# 在 mediapipe_gesture_recognizer.py 中
if gesture_name == "OK_Gesture":
    confidence *= 1.3  # 提高 30%
```

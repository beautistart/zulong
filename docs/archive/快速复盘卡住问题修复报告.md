# 快速复盘卡住问题修复报告

**修复日期**: 2026-04-05  
**问题**: 快速复盘卡在创建临时缓冲区  
**根本原因**: 事件循环死锁  
**修复状态**: ✅ 已完成

---

## 🔍 问题回顾

### 现象

快速复盘流程在创建临时缓冲区后完全卡住:

```
✅ [ReplayIntegration] 已进入复盘模式，会话 ID: c3d661fb
✅ [ReplayIntegration] 已同步状态到 state_manager
✅ [ReplayIntegration] 已创建临时缓冲区
❌ 之后没有任何日志... (卡死!)
```

### 根本原因

[`_get_recent_context()`](file:///d:/AI/project/zulong_beta4/zulong/review/integration.py#L736-L779) 方法中的事件循环冲突:

```python
# ❌ 问题代码
def _get_recent_context(self, minutes: int = 30):
    loop = asyncio.new_event_loop()  # 创建新事件循环
    asyncio.set_event_loop(loop)
    
    # ❌ 在已有事件循环的线程中调用 run_until_complete
    pool = loop.run_until_complete(SharedMemoryPool.get_instance())
    recent_texts = loop.run_until_complete(pool.get_recent(...))
    # → 死锁！
```

**死锁原理**:
```
Gatekeeper 线程 (已有事件循环)
    ↓
调用 _handle_user_active_review()
    ↓
调用 _get_recent_context()
    ↓
创建新事件循环 + run_until_complete()
    ↓
❌ 死锁！
   - 当前线程被阻塞
   - 异步任务需要事件循环调度
   - 但线程阻塞了，事件循环无法运行
   - → 任务永远无法完成
```

## 🔧 修复方案

### 修复 1: `_get_recent_context()` 改为异步方法

**文件**: `zulong/review/integration.py`  
**修改**: 第 736-779 行

```python
# ❌ 修复前：同步方法，使用 run_until_complete
def _get_recent_context(self, minutes: int = 30) -> Dict[str, Any]:
    loop = asyncio.new_event_loop()
    pool = loop.run_until_complete(SharedMemoryPool.get_instance())
    recent_texts = loop.run_until_complete(pool.get_recent(...))

# ✅ 修复后：异步方法，使用 await
async def _get_recent_context(self, minutes: int = 30) -> Dict[str, Any]:
    pool = await SharedMemoryPool.get_instance()
    recent_texts = await pool.get_recent(time_window_sec=minutes*60)
```

**关键改进**:
- ✅ 直接使用当前事件循环
- ✅ 使用 `await` 而非 `run_until_complete`
- ✅ 避免创建新事件循环
- ✅ 消除死锁风险

### 修复 4: Gatekeeper 调用方式改为异步

**文件**: `zulong/l1b/scheduler_gatekeeper.py`  
**修改**: 第 622-678 行

```python
# ❌ 修复前：同步调用
def _forward_to_replay_integration_immediate(self, text, priority):
    if review_type == 'quick':
        # ❌ 直接同步调用，导致死锁
        replay_integration._handle_user_active_review({...})

# ✅ 修复后：异步调用
def _forward_to_replay_integration_immediate(self, text, priority):
    import asyncio
    try:
        loop = asyncio.get_running_loop()
        # ✅ 在已有事件循环中创建异步任务
        asyncio.create_task(
            replay_integration._handle_user_active_review({...})
        )
    except RuntimeError:
        # ✅ 无事件循环时，创建新线程执行
        def run_async_task():
            new_loop = asyncio.new_event_loop()
            new_loop.run_until_complete(...)
        
        thread = threading.Thread(target=run_async_task)
        thread.start()
```

**关键改进**:
- ✅ 统一使用异步调用
- ✅ 有事件循环：`asyncio.create_task()`
- ✅ 无事件循环：创建新线程
- ✅ 完全消除死锁风险

## 📊 预判结果

### 情况 A: 视觉模型输出详细，但 L2 输出模糊

**日志示例**:
```
>>>>> 视觉模型原始输出 (RAW): 图像中显示一个人坐在书桌前，穿着黑色带有白色图案的外套...
>>>>> L2 最终原始输出 (RAW): 由于光线不足，我无法确定...
```

**结论**: L2 模型的 Prompt 有问题，导致过度谨慎。

**修复方案**: 修改 L2 的 Prompt，移除"如果看不清就说不知道"这类指令。

### 情况 B: 视觉模型本身就输出模糊

**日志示例**:
```
>>>>> 视觉模型原始输出 (RAW): 图像比较暗，看不清具体细节...
```

**结论**: 视觉模型真的没看清。

**可能原因**:
1. 图片太黑/光线不足
2. 图片模糊/对焦不准
3. 视觉模型的 Prompt 太谨慎

**修复方案**:
1. 检查摄像头画面质量
2. 调整视觉模型 Prompt，鼓励描述看到的内容
3. 增加图像预处理 (亮度、对比度增强)

### 情况 C: 视觉模型和 L2 输出都详细

**日志示例**:
```
>>>>> 视觉模型原始输出 (RAW): 一个人穿着黑色外套，外套上有白色几何图案...
>>>>> L2 最终原始输出 (RAW): 这个人穿着一件黑色带有白色图案的外套...
```

**结论**: 系统工作正常！

**下一步**: 检查是否有其他地方的过滤逻辑 (如 TTS 前的文本清洗) 修改了输出。

## 🔧 调试技巧

### 1. 增加日志级别

如果看不到日志，确保日志级别设置为 INFO 或 DEBUG:

```python
import logging
logging.basicConfig(level=logging.INFO)
```

### 2. 导出完整日志

```bash
python -m zulong.bootstrap 2>&1 | tee zulong_debug.log
```

然后在 `zulong_debug.log` 中搜索 `>>>>>`。

### 3. 对比测试

进行多次测试，对比不同场景:
- 明亮环境 vs 昏暗环境
- 静止物体 vs 移动物体
- 近景 vs 远景

## 🎯 预期修复方向

根据诊断结果，可能需要修改:

### 1. 视觉模型 Prompt (位置：`_analyze_keyframes`)

**当前**:
```python
prompt="请详细描述这张图像：场景、物体、颜色、形状等。"
```

**优化**:
```python
prompt="""请详细描述这张图像：
- 场景：室内/室外，什么房间
- 物体：人、家具、物品
- 颜色：主要颜色，衣服颜色
- 形状：物体形状，排列方式
- 文字：如果有任何可见文字，请读出

要求：
- 直接描述看到的内容
- 不要说"可能"、"如果"等不确定词汇
- 即使不确定，也描述最可能的情况"""
```

### 2. L2 综合 Prompt (位置：`_analyze_keyframes`)

**当前**:
```python
final_prompt = f"""基于以下视觉分析结果，回答用户问题。
...
"""
```

**优化**:
```python
final_prompt = f"""基于以下视觉分析结果，回答用户问题。

视觉分析：{vision_response}

用户问题：{query}

要求：
- 直接根据视觉分析结果回答
- 不要添加"可能"、"也许"等不确定词汇
- 如果视觉分析提到了具体细节，直接使用
- 除非视觉分析明确说"看不清"，否则不要说无法确定"""
```

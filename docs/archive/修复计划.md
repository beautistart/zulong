## 已完成：记忆节点与任务图谱的地址继承

### 概念说明

架构图中黄色虚线是**概念性区域划分**（宏空间 vs 任务空间），不是物理分割。整张图是连在一起的，地址继承就是让子节点携带完整的父级路径。

### 地址格式（像文件系统路径）

```
Session 根节点：  dialogue:session_abc123
Round 节点：      dialogue:session_abc123/dialogue:round_xxx
任务根节点：      dialogue:session_abc123/task:tg_xxx
任务子节点：      dialogue:session_abc123/task:tg_xxx/o1
任务孙节点：      dialogue:session_abc123/task:tg_xxx/o1_1
```

从任意子节点通过 `metadata["full_path"]` 都能追溯到完整上级路径。

### 已实现的文件和修改

| 文件 | 修改内容 |
|------|----------|
| `zulong/memory/graph_adapters.py` | `add_round()` 建立 round → task 的 REFERENCE 边 |
| `zulong/memory/graph_adapters.py` | `_link_round_to_session()` 更新 round 的 full_path，调用地址传播 |
| `zulong/memory/graph_adapters.py` | `_propagate_address_to_tasks()` 传播地址到关联任务节点 |
| `zulong/memory/graph_adapters.py` | `_propagate_address_to_task_children()` 递归传播到子任务 |
| `zulong/memory/graph_adapters.py` | `bind_session_to_task()` 更新任务节点地址并传播 |
| `zulong/memory/graph_adapters.py` | `TaskGraphAdapter.sync()` 从任务根节点继承完整路径到子节点 |
| `zulong/tools/task_tools.py` | `TaskCreatePlanTool` 查找最近 round，继承其路径 |
| `zulong/tools/task_tools.py` | `TaskAddNodeTool` 同步子节点到 MemoryGraph，继承父节点路径 |
| `zulong/l2/inference_engine.py` | 添加 `_active_task_graph_id` 跟踪，工具执行时记录 |

### 地址继承流程

```
1. Gatekeeper 创建 round（session_id=None）
   → add_round() 创建 round 节点，建立 REFERENCE 边到 task（如果存在）
   
2. L2 推理 → FC 循环 → task_create_plan
   → 查找最近 round，继承其路径创建任务根节点
   → 建立双向 REFERENCE 边
   
3. L2 _update_memory_async() → assign_session_by_similarity()
   → _link_round_to_session() 分配 session
   → _propagate_address_to_tasks() 传播地址到关联任务
   → _propagate_address_to_task_children() 递归传播到子任务

4. 用户调用 task_add_node
   → 查找父节点 full_path
   → 子节点地址 = 父路径/子ID
   → 同步到 MemoryGraph
```

### REFERENCE 边（双向关联）

```
dialogue:session_xxx/dialogue:round_xxx ──REFERENCE──> dialogue:session_xxx/task:tg_xxx
         ▲                                                      │
         │                        BFS 扩散可跨类型传播           │
         └──────────────────────────────────────────────────────┘
```

有了 REFERENCE 边，BFS 扩散激活可以从对话节点扩散到任务节点，反之亦然。

## P0 修复：上下文丢失

### 通俗解释根因

用户第 1 轮说"帮我做一个3D赛车游戏"，第 2 轮说"现在就开始帮我做"。
第 2 轮检索时，MemoryGraph 用**关键词匹配**去热窗口节点里找：

```
第 2 轮输入："技术由你推荐，现在就开始帮我做"
第 1 轮节点 label："对话轮次 round_xxx"
第 1 轮节点 metadata["goal"]："帮我做一个3D赛车游戏，多面体风格，色彩梦幻"

关键词匹配："技术由你推荐..." in "帮我做一个3D游戏..." → 不匹配 ❌
bigram 重叠率：不到 20% → 返回 0 ❌
FAISS 检索：round 节点可能没被索引 → 空 ❌
```

**结果**：检索返回空，LLM 收到的 system prompt 里没有第 1 轮内容，当作全新对话处理。

### 修复方案：热数据检索时沿 TEMPORAL 边追溯最近 N 轮

**改动位置**：`zulong/memory/memory_graph.py` 的 `_retrieve_hot()` 方法

#### 步骤 1：新增 `_trace_temporal_chain()` 方法

```python
def _trace_temporal_chain(self, top_n: int = 4) -> List[GraphNode]:
    """沿时间线获取最近的 N 个 round 节点，不依赖关键词匹配
    
    确保同一 session 内的最近对话始终被注入上下文。
    """
    rounds = []
    for node in self._nodes.values():
        if node.node_type == NodeType.DIALOGUE and node.metadata.get("sub_type") == "round":
            rounds.append(node)
    
    if not rounds:
        return []
    
    rounds.sort(key=lambda n: n.created_at, reverse=True)
    return rounds[:top_n]
```

#### 步骤 2：在 `_retrieve_hot()` 的 BFS 扩散后追加 TEMPORAL 追溯

```python
# 3. TEMPORAL 边追溯：沿时间线获取最近 N 轮的对话节点
recent_rounds = self._trace_temporal_chain(top_n=4)
for round_node in recent_rounds:
    if any(r["node_id"] == round_node.node_id for r in results):
        continue
    temporal_score = 0.4 * round_node.activation + 0.3
    if session_id and round_node.metadata.get("session_id") == session_id:
        temporal_score += 0.3
    results.append(
        self._node_to_result(round_node, temporal_score, "temporal_trace")
    )
```

#### 预期效果

- 第 2 轮检索时，TEMPORAL 边追溯直接拿到最近 4 轮对话
- 关键词匹配 + 时间线追溯双保险，上下文不再丢失
- 配合已实现的地址继承，对话节点与任务节点可互相追溯

## P2 修复：自主动态注意力机制断裂

### 通俗解释根因

系统里有**两套"注意力焦点"管理系统**，它们应该同步但实际没有。

#### 系统 A：MemoryGraph 的焦点路径（地图标记）

- 调用 `navigate_attention` 工具说"深入看细节"或"退回来看全局"
- 更新 `MemoryGraph._last_focus_context`
- 每次 LLM 推理时，焦点路径被注入到 system prompt
- **好比**：手里拿着一张**地图标记**，告诉 LLM "你现在站在图的哪个位置"

#### 系统 B：AttentionWindowManager 的焦点（记忆筛选器）

- 上下文窗口管理器，负责决定哪些历史消息保留、哪些丢弃
- 根据焦点给消息打分：焦点附近的消息分值高（保留），远的分值低（丢弃）
- **好比**：有一个**记忆筛选器**，只记住焦点附近的事

#### 断裂在哪里？

```
调用 navigate_attention 工具
    ↓
系统 A（地图标记）更新了 ✅
    ↓
系统 B（记忆筛选器）不知道 ❌
    ↓
筛选器还是按旧的焦点评分
    → 该保留的消息被扔掉了
```

**生活比喻**：眼睛看左边，耳朵还朝右边。

### 修复方案

#### 阶段 1：同步两套焦点系统

**改动位置**：`zulong/tools/attention_tool.py`

在 `NavigateAttentionTool.execute()` 中同步到 `AttentionWindowManager`。

#### 阶段 2：FC 循环集成窗口管理

**改动位置**：`zulong/l2/inference_engine.py` 的 FC 循环部分

在每轮开始时应用 `AttentionWindowManager.apply_window()` 剪裁 messages。

#### 阶段 3：System Prompt 添加注意力导航引导

在 system_parts 中添加 `navigate_attention` 工具的使用引导。

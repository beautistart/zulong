### 问题 1：快速复盘后无法与 L2 对话

**审查路径**:
```
用户输入 → Gatekeeper (scheduler_gatekeeper.py)
         → 检查 L2 状态 (state_manager.py)
         → 转发到 ReplayIntegration (integration.py)
         → 执行快速复盘 (_handle_quick_review_async)
         → 重置 L2 状态 (关键修复点)
```

**关键文件**:
1. `zulong/l1b/scheduler_gatekeeper.py` - 第 591 行 `_handle_review_mode_input`
2. `zulong/review/integration.py` - 第 289 行 `_handle_quick_review_async`
3. `zulong/core/state_manager.py` - 第 70 行 `set_l2_status`

### 问题 2：重复提示复盘模式选择

**审查路径**:
```
用户说"快速复盘" → Gatekeeper 检测关键词

                → _handle_user_active_review 检查 review_type
```

## 📝 审查要点

### 状态管理审查

1. **L2 状态流转**: 检查 `REVIEW_ANALYZING` → `IDLE` 的转换是否正确
2. **上下文同步**: 检查 `review_mode` 和 `review_type` 在 Gatekeeper 和 ReplayIntegration 之间的同步
3. **防重复机制**: 检查 `_review_processing` 标志是否有效

### 异步处理审查

1. **事件循环**: 检查异步调用是否正确使用 `asyncio.create_task` 或 `asyncio.run`
2. **线程安全**: 检查跨线程调用是否使用 `threading.Thread` 或 `loop.call_soon_threadsafe`
3. **死锁预防**: 检查是否有循环等待或资源竞争

### 错误处理审查

1. **异常捕获**: 检查所有 `try-except` 块是否正确记录日志
2. **状态恢复**: 检查异常情况下是否正确重置状态
3. **降级策略**: 检查失败时是否有合理的降级处理

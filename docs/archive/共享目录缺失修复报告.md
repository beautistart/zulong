# 共享目录缺失问题修复

**日期**: 2026-03-26  
**问题**: `shared_vision` 目录不存在，导致文件无法保存  
**状态**: ✅ 已修复

---

## 📋 问题诊断

### 现象

运行诊断工具时，发现：
```
⏰ [  0.0s] 当前状态:
   - latest_frame.jpg: 不存在
   - recent_clip.mp4: 不存在
⏰ [  5.0s] 当前状态:
   - latest_frame.jpg: 不存在
   - recent_clip.mp4: 不存在
...
```

**持续 60 秒都没有文件生成**。

---

### 根本原因

检查发现：
```bash
# 目录不存在
d:\AI\project\zulong_beta4\zulong\l1a\shared_vision\  ❌
```

**问题**: 视觉系统尝试保存文件到不存在的目录，导致保存失败。

---

## ✅ 修复方案

### 修复 1: 创建共享目录

**执行命令**:
```powershell
New-Item -ItemType Directory -Path "d:\AI\project\zulong_beta4\zulong\l1a\shared_vision" -Force
```

**结果**:
```
Directory: D:\AI\project\zulong_beta4\zulong\l1a
d----   2026/3/26 星期四  上午 0:0    shared_vision
```

✅ **目录已创建**

---

### 修复 2: 添加目录自动创建逻辑

**建议修改**: 在 `vision_short_term_memory.py` 的 `__init__` 方法中添加：

```python
def __init__(self, duration: int = 5, fps: int = 30):
    """初始化短期记忆"""
    self.duration = duration
    self.fps = fps
    self.ring_buffer = []
    
    # 🎯 关键修复：确保共享目录存在
    self.shared_vision_dir = Path(__file__).parent / "shared_vision"
    self.shared_vision_dir.mkdir(exist_ok=True)  # 自动创建目录
    
    self.latest_frame_path = self.shared_vision_dir / "latest_frame.jpg"
    self.recent_clip_path = self.shared_vision_dir / "recent_clip.mp4"
    self.metadata_path = self.shared_vision_dir / "meta.json"
    
    # ... 其他初始化代码
```

## 🔧 故障排查

### 问题 1: 目录存在但仍然无法保存

**可能原因**: 权限问题

**解决方案**:
```powershell
# 检查目录权限
Get-Acl zulong\l1a\shared_vision | Format-List
```

---

### 问题 2: 文件保存了但不更新

**可能原因**: 
- 视觉系统没有运行
- 运动检测阈值太高

**解决方案**:
1. 确认视觉系统正在运行
2. 查看日志是否有运动检测事件
3. 调整运动检测阈值

---

### 问题 3: 一直在保存（高频）

**可能原因**: 运动检测太敏感

**解决方案**:
```python
# 提高阈值
OpticalFlowMotionDetector(
    flow_threshold=5.0,
    motion_pixel_threshold=10000
)
```

## 🎓 经验总结

### 关键教训

1. **目录初始化很重要**
   - 共享目录必须在系统启动时创建
   - 不能假设目录已经存在

2. **自动创建优于手动创建**
   - 代码中自动创建更可靠
   - 避免部署时遗漏

3. **测试工具帮助快速定位**
   - `test_quick_save.py` 快速验证基本功能
   - `test_save_diagnosis.py` 深入分析保存行为

### 最佳实践

1. ✅ 在 `__init__` 中创建必要目录
2. ✅ 使用 `mkdir(exist_ok=True)` 安全创建
3. ✅ 保存前检查目录和文件权限
4. ✅ 提供清晰的错误信息

# 保存逻辑冲突问题修复

**日期**: 2026-03-25  
**问题**: 一直不停保存回溯视频，latest_frame.jpg 一直不保存  
**状态**: 🔧 修复中

---

## 📋 问题现象

用户报告：
> "现在，一直不停的保存回溯视频（不管画面是否变动），latest_frame.jpg 一直不保存，不更新"

---

## 🔍 问题分析

### 问题 1: 双重保存逻辑冲突

系统中有**两个地方**在保存 `latest_frame.jpg`：

1. **vision_processor.py** (事件驱动)
   ```python
   # 检测到运动时保存
   if motion_detected:
       self.vision_node.short_term_memory.save_latest_frame()
   ```

2. **vision_short_term_memory.py** (后台线程)
   ```python
   # 后台线程定期保存
   if self.motion_detected:
       self.save_latest_frame()
   ```

**问题**: 两个逻辑可能互相干扰，导致保存时机混乱。

---

### 问题 2: 运动检测过于敏感

如果运动检测阈值设置得太低，会导致：
- 轻微画面变化就触发保存
- 看起来像"一直在保存"
- 实际上是频繁检测到"运动"

---

### 问题 3: 日志级别不足

之前的日志是 `DEBUG` 级别，看不到详细信息：
```python
logger.debug(f"📸 事件驱动：已保存最新帧到 latest_frame.jpg")
```

需要改为 `INFO` 级别才能看到。

---

## ✅ 修复方案

### 修复 1: 调整后台线程频率

**文件**: [`vision_short_term_memory.py`](file:///d:/AI/project/zulong_beta4/zulong/l1a/vision_short_term_memory.py#L514-L548)

**修改**:
- 定期保存间隔：10 秒 → **30 秒**
- 避免干扰事件驱动保存

```python
# 静止时降低保存频率（每 30 秒一次，而不是每秒）
# 这样可以避免干扰 vision_processor 的事件驱动保存
elif (current_time - self.last_save_time) > 30.0:
    logger.info(f"👁️ [定期] 静止状态，保存最新帧 (30 秒间隔)")
    self.save_latest_frame()
    self.last_save_time = current_time
```

---

### 修复 2: 增强日志输出

**文件**: [`vision_processor.py`](file:///d:/AI/project/zulong_beta4/zulong/l1a/vision_processor.py#L225)

**修改**:
```python
logger.info(f"📸 [事件驱动] 已保存最新帧到 latest_frame.jpg (运动像素={motion_pixels})")
```

**效果**:
- 显示运动像素数量
- 区分事件驱动和定期保存

## 🎓 经验总结

### 关键教训

1. **避免双重保存逻辑**
   - 两个地方保存同一个文件容易冲突
   - 应该只有一个"真理源"

2. **日志标签很重要**
   - 清晰的标签帮助快速定位问题
   - `[事件驱动]` vs `[定期]`

3. **监控工具必不可少**
   - 没有监控很难发现问题
   - 诊断工具帮助快速定位

### 最佳实践

1. ✅ 关键操作添加日志标签
2. ✅ 定期保存频率不宜过高
3. ✅ 事件驱动优先于定时保存
4. ✅ 提供诊断工具帮助调试

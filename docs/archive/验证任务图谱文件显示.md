### 2. 文件打开功能 (index.html)

位置：`openclaw_bridge/web/static/index.html#L2487-L2508`

```javascript
function openFileInIDE(filePath) {
    console.log('[Web] 请求打开文件:', filePath);
    
    // 方案 1: 调用后端 API (如果 Web 服务器支持)
    fetch('/api/open-file', {
        method: 'POST',
        headers: {'Content-Type': 'application/json'},
        body: JSON.stringify({path: filePath})
    }).then(function(res) {
        if (res.ok) {
            console.log('[Web] 文件已在 IDE 中打开');
        } else {
            // 降级方案：直接在新窗口打开
            window.open('file://' + filePath, '_blank');
        }
    }).catch(function(err) {
        console.warn('[Web] API 调用失败，尝试直接打开:', err);
        window.open('file://' + filePath, '_blank');
    });
}
```

**功能说明**:
- ✅ 优先尝试调用后端 `/api/open-file` 接口
- ✅ API 失败时降级为 `file://` 协议直接打开
- ✅ 支持跨浏览器兼容

## ❓ 故障排查

### 问题 1: 看不到"关联文件"区域

**可能原因**:
1. 浏览器缓存了旧版本的 HTML
2. 节点数据中没有 `files` 字段

**解决方法**:
```bash
# Chrome/Edge: Ctrl+F5 (Windows) 或 Cmd+Shift+R (Mac)
# Firefox: Ctrl+F5 或 Ctrl+Shift+R
# Safari: Cmd+Option+E 然后 Cmd+R
```

### 问题 2: 点击文件没反应

**检查方法**:
1. 打开浏览器开发者工具 (F12)
2. 查看 Console 是否有错误信息
3. 检查 Network 标签页是否有 `/api/open-file` 请求

**可能原因**:
- JavaScript 错误阻止了点击事件
- 文件路径包含特殊字符需要转义

### 问题 3: 文件无法打开

**可能原因**:
- 现代浏览器阻止 `file://` 协议打开本地文件 (安全限制)
- 文件路径不存在或已被移动

**解决方法**:
1. 实现后端 `/api/open-file` 接口
2. 或者在浏览器设置中允许本地文件访问 (不推荐)

在 Web 服务器中添加 `/api/open-file` 端点:

```python
@app.post("/api/open-file")
async def open_file(data: dict):
    import subprocess
    import sys
    
    file_path = data.get("path")
    
    # 验证文件是否存在
    if not os.path.exists(file_path):
        return {"success": False, "error": "文件不存在"}
    
    # 使用系统默认应用打开文件
    if sys.platform == "win32":
        os.startfile(file_path)
    elif sys.platform == "darwin":
        subprocess.run(["open", file_path])
    else:
        subprocess.run(["xdg-open", file_path])
    
    return {"success": True}
```

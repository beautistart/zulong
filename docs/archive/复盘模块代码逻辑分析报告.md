## 🔥 核心问题分析

### 问题 1: 快速复盘流程中缺少 L2 交互

**代码位置**: `zulong/review/integration.py` 第 224-290 行

**快速复盘流程**:
```python
def _handle_quick_review(self, recent_data, context):
    # 1. 分析对话
    analysis_result = self._analyze_conversation_with_l2(recent_data)  # ✅ 调用 L2
    
    # 2. 生成经验
    experiences = self._generate_experiences(analysis_result)  # ❌ 未调用 L2
    
    # 3. 应用经验
    self._apply_experiences(experiences)  # ❌ 未调用 L2
    
    # 4. 生成报告
    report = self._generate_review_report(...)  # ❌ 未调用 L2
    
    # 5. 发布结果
    self._publish_l2_response(response_text)  # ✅ 发布响应
```

**问题根源**:
- ✅ `_analyze_conversation_with_l2()` - **调用了 L2**
- ❌ `_generate_experiences()` - **未调用 L2，直接基于分析结果生成**
- ❌ `_apply_experiences()` - **未调用 L2，直接保存到 RAG**
- ❌ `_generate_review_report()` - **未调用 L2，直接拼接文本**

**对比深度复盘**:
```python
async def _handle_deep_review(self, recent_data, context):
    # 1. 检索记忆
    memory_data = self._retrieve_long_term_memory(recent_data)  # ✅ 调用 RAG
    
    # 2. L2 多维分析
    analysis = self._deep_analysis_with_l2(memory_data)  # ✅ 调用 L2
    
    # 3. L2 生成经验草案
    experiences = self._generate_experience_drafts_with_l2(analysis)  # ✅ 调用 L2
    
    # 4. 发布草案，等待用户确认
    self._publish_experience_drafts(experiences)  # ✅ 发布到 L2
```

**结论**: **快速复盘为了"快速",牺牲了 L2 的深度参与，导致用户体验下降!**

### 问题 2: Gatekeeper 直接调用 vs 事件驱动

**代码位置**: `zulong/l1b/scheduler_gatekeeper.py` 第 622-670 行

**当前实现**:
```python
def _forward_to_replay_integration_immediate(self, text, priority):
    replay_integration = get_replay_integration()
    
    if review_type == 'quick':
        # ❌ 直接同步调用方法
        replay_integration._handle_user_active_review({
            'user_input': text,
            'review_type': 'quick'
        })
    else:
        # ✅ 异步调用
        asyncio.create_task(self._handle_deep_review_async(...))
```

**问题分析**:
1. **直接方法调用** vs **事件驱动**
   - 直接调用：耦合度高，难以调试
   - 事件驱动：解耦，可追踪，易于扩展

2. **同步 vs 异步**
   - 快速复盘：同步调用，阻塞主线程
   - 深度复盘：异步调用，不阻塞

3. **状态不一致**
   - Gatekeeper 设置了 `review_mode=True`
   - 但 ReplayIntegration 内部又设置了一次 `self.review_mode = True`
   - 可能导致状态不同步

**建议架构**:
```python
# Gatekeeper 只发布事件，不直接调用方法
def _forward_to_replay_integration_immediate(self, text, priority):
    # ✅ 发布 REPLAY_REQUESTED 事件
    event = ZulongEvent(
        type=EventType.REPLAY_REQUESTED,  # 新增事件类型
        source="Gatekeeper",
        payload={
            'review_type': 'quick',
            'user_input': text
        },
        priority=priority
    )
    event_bus.publish(event)
```

    """🔥 新增：订阅 REPLAY_REQUESTED 事件"""
    event_bus.subscribe(
        EventType.REPLAY_REQUESTED,  # 🔥 新增
        self.on_replay_requested,  # 🔥 新增处理方法
        "ReplayIntegration"
    )

async def on_replay_requested(self, event):
    """🔥 新增：处理复盘请求事件"""
    review_type = event.payload.get('review_type')
    user_input = event.payload.get('user_input')
    
    if review_type == 'quick':
        self._handle_quick_review_with_l2(user_input)  # 🔥 增强版
    elif review_type == 'deep':
        await self._handle_deep_review_with_l2(user_input)

def _handle_quick_review(self, recent_data, context):
    try:
        # 🔥 新增：设置状态为 REVIEW_ANALYZING

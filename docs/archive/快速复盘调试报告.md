## 🔍 问题分析

### 可能的根本原因

1. **新线程中的异步任务执行失败**
   - 新线程创建了事件循环
   - 但执行 `_handle_user_active_review()` 时出错
   - **错误没有被捕获和记录**

2. **SharedMemoryPool 跨线程访问问题**
   - 主线程创建的 SharedMemoryPool 单例
   - 新线程可能无法正确访问

3. **状态管理器跨线程问题**
   - `state_manager` 可能不是线程安全的
   - 跨线程访问可能导致死锁

## 🔧 已实施的修复

### 修复 1: 增加详细日志

在 Gatekeeper 的线程执行函数中添加了详细日志:

```python
def run_async_task():
    new_loop = asyncio.new_event_loop()
    asyncio.set_event_loop(new_loop)
    try:
        logger.info(f"[Gatekeeper][线程] 新事件循环已创建")
        logger.info(f"[Gatekeeper][线程] 开始执行异步任务...")
        new_loop.run_until_complete(
            replay_integration._handle_user_active_review({...})
        )
        logger.info(f"[Gatekeeper][线程] ✅ 异步任务执行完成")
    except Exception as inner_e:
        logger.error(f"[Gatekeeper][线程] ❌ 异步任务执行失败：{inner_e}", exc_info=True)
    finally:
        logger.info(f"[Gatekeeper][线程] 关闭事件循环")
        new_loop.close()
```

### 修复 2: 等待线程启动

添加了 0.1 秒延迟，确保线程有时间启动:

```python
thread.start()
logger.info(f"[Gatekeeper] ✅ 已创建线程 (线程 ID: {thread.ident})")
time.sleep(0.1)  # 等待线程启动
logger.info(f"[Gatekeeper] 线程已启动，等待执行...")
```

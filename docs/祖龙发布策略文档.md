## 二、策略总纲：分层披露 + 时间差保护 + 叙事先行

**核心原则**: 先建立认知霸权，再释放代码。让所有人第一次听到"图记忆+赫布学习+注意力导航"这套叙事时，脑子里的关联词就是"祖龙"。

---

## 三、叙事定位

### 核心故事线
> "一个室内设计师，用 AI 助手在不到一个月内写出了 82000 行代码，造了一个有记忆、会遗忘、能自主调整注意力的 AI 大脑。"

### 三圈层击穿策略
- **AI 技术圈**: 图记忆 + 赫布学习 + 注意力控制的工程实现
- **创业圈**: 非技术背景如何用 AI 完成硬核系统级项目
- **大众圈**: AI 的"记忆会衰减、常用的记住、不用的忘掉"很有人情味

### 双定位分场景宣传
- **对机器人圈**: "具身智能大脑" -- 多模态感知 + 任务规划 + 长期记忆 + 中断恢复
- **对 AI Agent 圈**: "认知操作系统" -- 记忆管理 + 注意力调度 + 任务编排
- **对投资人**: "底层是认知操作系统，上层可以接机器人(硬件)也可以接纯软件 Agent"

## 四、代码保护策略

| 内容 | 策略 | 许可证 |
|------|------|--------|
| 旧版本代码(已发) | 保持不变，作为"入门版"引流 | AGPL-3.0 + MIT(插件) |
| **新版本核心代码** | **不发 GitHub，闭源保留** | 不适用 |
| 架构设计文档 | 开源发布 | CC BY-NC-SA 4.0 |
| 深度分析报告 | 可公开发布(展示深度) | CC BY-NC-SA 4.0 |
| API/插件接口规范 | 开源(吸引开发者生态) | MIT |

### AGPL-3.0 的保护效力
- 任何人拿代码做 SaaS 服务，必须开源全部代码
- 大公司法务部门看到 AGPL 就会放弃直接使用
- 要么不用，要么来谈商业授权 -- 这就是商业化入口

## 七、商业化路径

### 推荐路径: 技术授权/SDK 为主，融资扩团队为辅

**短期**: 将祖龙认知架构 SDK 化，授权给机器人公司/AI Agent 公司
- 卖的是"认知记忆系统"这个能力

**中期**: 拿到投资后扩大团队
- 创始人角色: 架构师/产品设计者
- 招聘方向: Python 工程师(性能优化、测试、部署)
- 精锐团队 3-5 人足够

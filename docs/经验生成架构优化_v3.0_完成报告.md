```python
async def _auto_extract_experience(self):
    """自动提取经验 (解决断层 1)"""
    logger.info("[InferenceEngine] 开始自动提取经验...")
    
    # 1. 从对话历史中提取
    candidates = self.experience_generator.extract_from_dialogue(
        self.conversation_history
    )
    
    # 2. 过滤低置信度
    high_confidence = [c for c in candidates if c.confidence >= 0.6]
    
    # 3. 调用存储层的 add_experience (包含去重逻辑)
    success_count = 0
    for candidate in high_confidence:
        success = self.experience_generator.add_experience_to_rag(candidate)
        if success:
            success_count += 1
    
    logger.info(f"[InferenceEngine] 成功保存 {success_count} 条经验")
```

```python
def add_experience(self, content: str, category: str, 
                  importance: str = "must_learn",
                  domain: str = "general") -> str:
    # 🔥 v3.0 新增：存储层去重检查 (原子操作)
    if self._is_duplicate(content, threshold=0.95):
        logger.warning(f"⚠️ [去重拦截] 发现高度相似经验，已跳过写入。内容：{content[:50]}...")
        return ""  # 返回空字符串表示未写入
    
    doc = RAGDocument(
        content=content,
        metadata={"category": category},
        importance=importance,
        domain=domain
    )
    return self.add_document(doc)
```

**去重策略**:
1. **SimHash 快速过滤**: 海明距离<3 视为相似
2. **向量相似度精确比对**: 余弦相似度>0.95 确认为重复
3. **指纹缓存**: 避免重复计算 SimHash

**效果**:
- ✅ **多入口防御**: 无论经验来自实时纠错、复盘还是外部导入，都拦截
- ✅ **原子性保证**: "检查 + 写入" 原子操作，避免并发问题
- ✅ **高性能**: SimHash 快速过滤 + 向量相似度精确比对
- ✅ **可配置阈值**: 默认 95% 相似度，可根据需求调整

## 🔥 核心升级亮点

### 1. 存储层去重机制

**为什么重要**:
- 经验可能来自**多个入口**: 实时纠错、复盘、外部导入
- 如果只在**应用层**去重，当**外部导入**重复经验时，系统就会重复写入
- **写入操作必须是原子的**: "检查 + 写入" 逻辑必须在存储层完成

**v3.0 方案**:
```python
# EnhancedExperienceStore.add_experience()
def add_experience(self, experience: ExperienceCandidate) -> bool:
    # 🔴 第一步：强去重检查 (基于内容指纹)
    if self._is_duplicate(experience.content, threshold=0.95):
        return False  # 拦截重复
    
    # ✅ 第二步：原子写入
    self.vector_db.add(...)
    self.experience_db.insert(...)
    return True
```

**效果**:
- ✅ **防御性无敌**: 无论经验来自哪里，相似度>95% 直接拦截
- ✅ **解耦**: 复盘模块不再需要知道纠错模块发生了什么
- ✅ **数据纯净**: RAG 向量库中永远只有一条真理

### 2. L2 主导的复盘状态机

**核心思想**:
- L2 通过状态机管理复盘流程
- L1-B 简单转发，不越权处理
- 支持多轮交互和灵活扩展

**状态流转**:
```
IDLE → REVIEW_WAITING → REVIEW_ANALYZING → IDLE
```

**效果**:
- ✅ 流程清晰，状态可控
- ✅ 支持中断和恢复
- ✅ 易于调试和监控

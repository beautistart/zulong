## 🔍 核心问题分析

### 问题 1: 特征值范围未知

**现状**:
- 使用 `np.mean(np.abs(features))` 计算特征能量
- 但 features 的范围是多少？
  - MobileNetV3 输出：[-0.3656, 1.5046] (未经过激活)
  - 经过特征提取后：未知

**影响**:
- 如果特征能量普遍 < 0.02，则所有动作都会被分类为 STILL
- 如果特征差异普遍 < 0.008，则无法区分挥手和注视

**验证方法**:
```python
# 在 classify_action 中打印特征统计
logger.info(f"Feature range: [{features.min():.4f}, {features.max():.4f}]")
logger.info(f"Feature energy: {feature_energy:.6f}")
logger.info(f"Feature diff: {feature_diff:.6f}")
```

### 问题 2: 规则分类阈值不合理

**当前阈值**:
```python
high_energy_threshold = 0.05
low_energy_threshold = 0.02
high_diff_threshold = 0.015
low_diff_threshold = 0.008
```

**实测数据** (95.9 秒):
- 平均分数：0.193
- 最高分数：0.573
- 分数范围：0.000 - 0.573

**分析**:
- 平均分数 0.193 远低于阈值 0.25
- 说明大部分特征能量/差异都低于阈值
- **阈值可能设置过高**

### 问题 3: 分数计算公式问题

**当前公式**:
```python
# WAVING
intent_score = min(1.0, 0.75 + (feature_energy + feature_diff) * 0.25)

# APPROACHING
intent_score = min(1.0, 0.7 + feature_energy * 0.3)

# GAZING
intent_score = min(1.0, 0.7 + feature_diff * 0.3)

# STILL
intent_score = max(0.7, 1.0 - (feature_energy + feature_diff) * 0.3)
```

**问题**:
- 基础分数偏高 (0.7-0.75)
- 特征贡献偏低 (* 0.25-0.3)
- **导致分数区分度不够**

**举例**:
- 如果 feature_energy = 0.01, feature_diff = 0.005
- WAVING: 0.75 + (0.01 + 0.005) * 0.25 = 0.754
- 但实际观测到的平均分只有 0.193

**矛盾**:
- 代码计算应该是 0.75+
- 实测却是 0.193
- **说明代码逻辑有问题！**

## 🐛 根本原因定位

### 代码审查发现

查看 `action_classifier.py` Line 397-420:

```python
# Line 397-420: 规则分类逻辑
if feature_energy > high_energy_threshold and feature_diff > high_diff_threshold:
    intent_type = "WAVING"  # 挥手
    intent_score = min(1.0, 0.75 + (feature_energy + feature_diff) * 0.25)
elif feature_energy > low_energy_threshold and feature_diff <= high_diff_threshold:
    intent_type = "APPROACHING"  # 靠近
    intent_score = min(1.0, 0.7 + feature_energy * 0.3)
elif feature_energy <= low_energy_threshold and feature_diff > low_diff_threshold:
    intent_type = "GAZING"  # 注视
    intent_score = min(1.0, 0.7 + feature_diff * 0.3)
else:
    intent_type = "STILL"  # 静止
    intent_score = max(0.7, 1.0 - (feature_energy + feature_diff) * 0.3)
```

**问题**:
1. 所有分支的基础分数都是 0.7+
2. 但实测平均分只有 0.193
3. **说明大部分情况进入了 else 分支 (STILL)**

**验证**:
- 如果 feature_energy <= 0.02 且 feature_diff <= 0.008
- 则进入 STILL 分支
- 分数 = max(0.7, 1.0 - (0.02 + 0.008) * 0.3) = max(0.7, 0.992) = 0.992

**仍然矛盾**:
- STILL 的分数应该是 0.7+
- 但实测是 0.193
- **说明还有其他代码逻辑！**

## 🎯 真正的问题

### 问题根源

**MobileNetV3 预训练模型不是为动作识别训练的！**

1. **模型用途不匹配**
   - MobileNetV3-ImageNet: 图像分类 (1000 类)
   - 动作识别：时空特征 (5 类意图)
   - **用图像分类模型做动作识别，置信度自然低**

2. **置信度计算方式**
   - ImageNet 模型输出 1000 类的概率
   - 映射到 5 类意图时，置信度会被稀释
   - **平均 0.193 符合预期**

3. **特征提取不充分**
   - 单帧图像 → 1280 维特征
   - 没有时序信息 (TSM 未实现)
   - **无法捕捉动作的动态特征**

### 方案 2: 修改分数映射 (中期)

**问题**: 模型 confidence 直接作为 intent_score 不合理

**解决**: 重新映射到 0-1 范围

```python
# 在 optimized_vision_processor.py 中
def _layer3_action_classification(self, frame):
    if self._model_loader and self._model_loader.mobilenet_model:
        action_label, confidence = self._model_loader.classify_action(frame)
        
        # 重新映射置信度
        # ImageNet 模型 confidence 通常 0.01-0.8
        # 映射到 0-1: (confidence - 0.01) / (0.8 - 0.01)
        intent_score = (confidence - 0.01) / 0.79
        intent_score = max(0.0, min(1.0, intent_score))  # 截断
        
        intent_type = action_label.upper()
        return intent_score, intent_type
```

## 一、项目概述

**祖龙 (ZULONG)** 是一个多层架构的智能体（Agent）系统，设计目标是构建具备感知、认知、记忆、学习和执行能力的类人智能系统。系统采用分层架构（L0-L3），从底层硬件感知到高层专家推理，形成一个完整的"感知-认知-行动"闭环。

### 核心特性
- **多层架构**: L0(设备层) → L1(感知层) → L2(认知推理层) → L3(专家层)
- **事件驱动**: 基于 EventBus 的异步事件总线
- **图记忆系统**: 基于 NetworkX 的统一记忆图谱，支持 BFS 扩散激活
- **RAG 增强**: 经验库、知识库、记忆库三库分离
- **技能包系统**: 可插拔的技能包运行时
- **复盘机制**: 任务失败后的自动归因分析和系统补丁生成
- **MCP 协议**: 通过 MCP Server 暴露记忆能力给外部 IDE（如灵码）

## 三、记忆系统（核心亮点）

### 3.1 MemoryGraph 记忆图谱

记忆图谱是祖龙系统的核心记忆基础设施，基于 NetworkX 有向图实现，支持：

- **异构节点类型**: task, dialogue, knowledge, experience, episode, file, concept, person, document
- **边类型**: hierarchy(父子), dependency(依赖), reference(引用), temporal(时间序列), semantic(语义), causal(因果), association(赫布关联)
- **BFS 扩散激活**: 从种子节点出发，沿边传播激活值，实现"从任意节点追溯全局关联"
- **赫布学习**: 共激活的边自动增强权重
- **突触修剪**: 基于艾宾浩斯遗忘曲线的边权衰减和孤立节点清理
- **FAISS 侧车索引**: 摘要向量检索，支持语义搜索
- **多维标签**: importance(重要度) + temperature(温度) + memory_strength(记忆强度)
- **JSON 持久化**: 原子写入（temp + rename），支持崩溃恢复

### 3.2 并行检索策略

```
检索请求
    │
    ├─→ 路径A: 热数据遍历 (最近30分钟)
    │    ├─ 当前会话节点注入
    │    ├─ 关键词匹配 (bigram 重叠)
    │    ├─ BFS 1跳扩散
    │    └─ 元记忆查询检测
    │
    └─→ 路径B: 冷数据混合检索
         ├─ FAISS 向量检索 (70% 权重)
         ├─ 关键词检索 (30% 权重)
         └─ BFS 下钻获取详情
         
合并 → 按分数排序 → Top-K 返回
```

### 3.3 三库 RAG 架构

| 库名 | 类型 | 用途 |
|------|------|------|
| ExperienceRAG | 经验库 | 存储历史任务经验、教训 |
| KnowledgeRAG | 知识库 | 存储领域知识、实体关系 |
| MemoryRAG | 记忆库 | 存储对话记忆、个人信息 |

## 四、L2 认知推理层

### 4.1 推理引擎 (Inference Engine)

- 支持 Ollama 和 vLLM 两种后端
- 核心模型 + 备份模型双实例
- Function Calling 循环，支持工具调用
- 流式输出支持

### 4.2 任务图谱 (Task Graph)

- 任务节点创建、依赖管理
- 任务冻结/恢复/归档
- 可恢复任务检查点

### 4.3 中断控制 (Interrupt Controller)

- L2 空闲时直连唤醒
- L2 忙碌时触发中断流程
- 优先级调度

## 九、关键设计决策

1. **事件驱动架构**: 所有模块通过 EventBus 通信，解耦各层
2. **单例模式**: MemoryGraph, SharedMemoryPool, StateManager 等核心组件使用单例
3. **防抖保存**: 记忆图谱变更 2 秒后自动落盘，避免频繁写入
4. **原子写入**: 先写临时文件再 rename，防止崩溃导致数据损坏
5. **双模型冗余**: L2 推理引擎支持核心模型 + 备份模型
6. **三层注意力**: L1-C 静默注意 → L1-A 反射 → L1-B 调度

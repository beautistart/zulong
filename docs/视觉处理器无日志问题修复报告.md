# 视觉处理器无日志问题修复报告

**问题发现日期**: 2026-04-02  
**问题描述**: Bootstrap 模式下视觉处理器没有日志输出，系统对用户语音输入无视觉响应  
**修复状态**: ✅ 已修复

---

## 🔍 问题分析

### 现象
1. 系统启动成功，所有模型加载正常
2. 用户输入"你能看见我吗"，系统无视觉响应
3. 视觉处理器没有输出任何日志

### 根本原因

`OptimizedVisionProcessor.feed_frame()` 方法中使用了 `asyncio.create_task()` 来异步处理帧：

```python
# ❌ 错误代码
def feed_frame(self, frame: np.ndarray, timestamp: float):
    # ...
    asyncio.create_task(self._process_frame_async(frame, timestamp))  # 问题在这里
```

**问题原因**:
- `asyncio.create_task()` 必须在有事件循环的异步上下文中才能工作
- Camera 是在**同步线程**中调用 `feed_frame()` 的
- 没有事件循环，`asyncio.create_task()` 会静默失败，导致帧处理逻辑从未执行

### 调用链
```
CameraDevice._capture_loop() [同步线程]
  └─> vp.feed_frame(frame, timestamp) [同步方法]
       └─> asyncio.create_task(_process_frame_async) [❌ 失败！无事件循环]
```

---

## 🔧 修复方案

### 方案选择

由于 Camera 是同步调用 `feed_frame()`，我们需要将异步处理改为同步处理：

**选项 1**: 使用 `asyncio.run()` 运行异步任务
```python
# 不推荐：每次调用都创建新的事件循环，性能差
asyncio.run(self._process_frame_async(frame, timestamp))
```

**选项 2**: 改为同步方法 ✅ (推荐)
```python
# 推荐：直接调用同步方法，性能最佳
self._process_frame_sync(frame, timestamp)
```

### 修复步骤

#### 1. 创建同步处理方法 `_process_frame_sync`

```python
def _process_frame_sync(self, frame: np.ndarray, timestamp: float):
    """
    同步处理帧 (核心逻辑)
    
    流程:
    1. Layer 1: YOLO-Nano 人体检测
    2. Layer 2: ROI 增益放大 + 像素差分
    3. Layer 3: MobileNetV4-TSM 动作分类
    4. Layer 4: 鹰眼模式 (Digital Zoom + MediaPipe)
    """
    try:
        # ========== Layer 1: 人体检测 ==========
        human_bboxes = self._layer1_human_detection(frame)
        
        if not human_bboxes:
            self.state_machine['layer1_state'] = 'NO_HUMAN'
            self.shared_memory['human_detected'] = False
            return
        
        # ... 后续处理逻辑
```

#### 2. 修改 `feed_frame` 方法

```python
def feed_frame(self, frame: np.ndarray, timestamp: float):
    """
    输入帧 (由 CameraDevice 主动推送)
    """
    try:
        if not self.is_running:
            return
        
        # 添加到缓冲区
        self.frame_buffer.append(frame.copy())
        self.timestamps.append(timestamp)
        
        # 保持缓冲区大小
        if len(self.frame_buffer) > self.frame_buffer.maxlen:
            self.frame_buffer.popleft()
            self.timestamps.popleft()
        
        # ✅ 修复：同步处理帧 (而非 asyncio.create_task)
        self._process_frame_sync(frame, timestamp)
        
    except Exception as e:
        logger.error(f"❌ [feed_frame] 错误：{e}", exc_info=True)
```

#### 3. 删除重复的异步方法

删除了旧的 `_process_frame_async` 方法，避免代码重复和维护困扰。

## 🎯 关键教训

### 1. 异步与同步的边界

**教训**: 不要在同步上下文中使用 `asyncio.create_task()`

**正确做法**:
- 如果调用方是同步的，被调用方也应该是同步的
- 如果必须调用异步方法，使用 `asyncio.run()` 或在异步上下文中调用

### 2. 摄像头驱动架构

**教训**: Camera 使用同步线程捕获帧，视觉处理器必须支持同步调用

**架构决策**:
```
CameraDevice [同步线程]
  └─> feed_frame() [同步方法]
       └─> _process_frame_sync() [同步处理] ✅
```

### 3. 日志的重要性

**教训**: 如果系统没有日志输出，说明代码路径没有执行

**调试技巧**:
- 在关键路径添加日志
- 检查日志级别是否正确
- 使用异常捕获并记录错误

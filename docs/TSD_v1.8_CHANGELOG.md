## ✅ 新增内容

### 1. 2.2.6 L1 层模块化插件架构 (新增章节)

在 **2.2.5 基础设施层** 后新增 **2.2.6** 章节，完整定义了 L1 层模块化插件架构:

#### 核心组件
- **IL1Module 接口**: 抽象基类，定义标准方法
  - `module_id`: 唯一标识符
  - `priority`: 优先级枚举
  - `initialize()`: 初始化
  - `process_cycle()`: 单周期处理
  - `health_check()`: 健康检查
  - `shutdown()`: 关闭清理

- **PluginManager**: 插件管理器
  - 动态加载/卸载插件
  - 优先级排序执行
  - 异常隔离机制
  - 性能统计监控

- **Shared Memory**: 共享内存字典
  - 插件间松耦合通信
  - 命名空间隔离
  - 读写规则约束

- **ZulongEvent**: 统一事件格式
  - 插件输出唯一方式
  - 标准化事件 Schema

#### 插件注册与发现
```yaml
# config/l1_plugins.yaml
plugins:
  - module_id: "L1A/Motor"
    class_path: "zulong.plugins.motor.l1a_motor_plugin.L1A_MotorPlugin"
    enabled: true
    priority: "HIGH"
    config:
      obstacle_threshold: 0.3
      max_speed: 0.8
```

#### 执行流程
```python
manager = PluginManager()
manager.load_plugins("config/l1_plugins.yaml")
manager.start()

while True:
    events = manager.run_cycle(shared_memory)
    for event in events:
        event_bus.publish(event)
```

#### 异常隔离机制
- 每个插件 `process_cycle` 包裹在 try-except 中
- 单个插件崩溃不影响其他插件
- 自动记录错误日志并生成 `SYSTEM_ERROR` 事件
- 健康检查连续失败 3 次后自动禁用插件

#### 优先级调度策略
| 优先级 | 执行顺序 | 使用场景 | 示例插件 |
|--------|---------|---------|---------|
| CRITICAL | 1 | 安全相关、紧急事件 | L1E/Gas |
| HIGH | 2 | 快速响应、反射动作 | L1A/Motor |
| NORMAL | 3 | 普通任务、感知分析 | L1C/Vision |
| LOW | 4 | 后台任务、数据同步 | L1F/Logger |

#### 共享内存键值规范
- 命名空间隔离 (`motor.*`, `vision.*`)
- 插件只能写入自己的命名空间
- 可以读取其他插件的命名空间 (只读)
- 禁止直接修改其他插件的数据

#### 标准插件列表
| 插件 ID | 优先级 | 职责 | 输入事件 | 输出事件 |
|--------|--------|------|---------|---------|
| L1A/Motor | HIGH | 电机控制、障碍反射 | SENSOR_OBSTACLE | ACTION_MOTOR |
| L1C/Vision | NORMAL | 视觉分析、运动检测 | SENSOR_VISION_REQUEST | VISION_DATA_READY |
| L1D/Voice | CRITICAL | 语音唤醒、命令识别 | USER_SPEECH | USER_WAKEWORD |
| L1E/Gas | CRITICAL | 气体检测、火灾报警 | SENSOR_GAS | ACTION_SPEAK |

#### 性能要求
- 插件加载时间 < 100ms
- 单插件 `process_cycle` 执行时间 < 10ms
- 总周期时间 < 33ms (30 FPS)
- 健康检查间隔 10 秒
- 异常恢复时间 < 1 秒

# 复盘机制快速参考指南

## 🚀 快速启动

### 1. 启动系统
```bash
python -m zulong.bootstrap
```

### 2. 启动复盘
```
用户：启动复盘
系统：请选择复盘模式：快速复盘 或 深度复盘
```

### 3. 选择模式
```
用户：快速复盘
系统：已进入快速复盘模式，现在我们可以自由对话了...
```

### 4. 进行对话
```
用户：今天学习了祖龙系统的复盘机制
系统：复盘机制是祖龙系统的核心功能之一...
```

### 5. 结束复盘
```
用户：结束复盘
系统：正在分析我们的对话...
系统：已生成 3 条经验，请说'确认'保存或'修改'重新对话
```

### 6. 确认经验
```
用户：确认
系统：✅ 复盘完成，已保存 3 条经验到记忆库
```

## 📋 关键命令

| 命令 | 作用 | 阶段 |
|------|------|------|
| `启动复盘` | 启动复盘向导 | 任意阶段 |
| `快速复盘` | 选择快速模式 | 模式选择阶段 |
| `深度复盘` | 选择深度模式 | 模式选择阶段 |
| `退出` / `取消` | 退出复盘 | 任意阶段 |
| `结束复盘` | 结束对话，提取经验 | 对话进行阶段 |
| `确认` / `保存` | 保存经验 | 经验确认阶段 |
| `修改` / `重新` | 重新开始对话 | 经验确认阶段 |

---

## 🔍 三阶段状态机

### 阶段 1：模式选择 (MODE_SELECTING)
- **状态**：等待用户选择快速或深度
- **接受的指令**：
  - `快速复盘` → 进入快速模式
  - `深度复盘` → 进入深度模式
  - `退出` / `取消` → 取消复盘

### 阶段 2：对话进行 (REVIEW_ACTIVE)
- **状态**：L2 正常对话，后台记录到缓冲区
- **接受的指令**：
  - `结束复盘` → 提取经验
  - `退出` → 退出复盘（不保存经验）
  - 其他输入 → 正常对话

### 阶段 3：经验确认 (EXPERIENCE_CONFIRMING)
- **状态**：展示生成的经验，等待用户确认
- **接受的指令**：
  - `确认` / `保存` → 保存到经验库
  - `修改` / `重新` → 重新开始对话
  - `退出` → 退出复盘（不保存经验）

## 🏗️ 核心组件

### ReviewTrigger（复盘触发器）
- **职责**：检测复盘触发条件
- **触发类型**：
  - 用户主动触发（高优先级）
  - 安静模式触发（中优先级）
  - 夜间定时触发（低优先级）

### ReplayIntegration（复盘集成器）
- **职责**：协调复盘流程
- **核心方法**：
  - `on_replay_triggered()` - 接收触发事件
  - `handle_mode_selection()` - 处理模式选择
  - `handle_end_review()` - 处理结束复盘
  - `handle_experience_confirmation()` - 处理经验确认

### ReviewStateManager（复盘状态管理器）
- **职责**：统一管理复盘状态
- **核心状态**：
  - `is_active()` - 是否处于复盘模式
  - `get_stage()` - 当前阶段
  - `get_mode()` - 复盘模式（快速/深度）

### ReviewTempBuffer（复盘临时缓冲区）
- **职责**：隔离复盘期间的对话
- **特点**：
  - 不污染主记忆池
  - 支持导出分析
  - 自动限制最大长度（100 条）

### ExperienceExtractor（经验提取器）
- **职责**：调用 L2 进行结构化分析
- **流程**：
  1. 构建系统提示词
  2. 调用 L2（vLLM）
  3. 解析 JSON 响应
  4. 验证数据结构

## 🐛 常见问题排查

### 问题 1：复盘无法启动

**症状**：输入"启动复盘"无响应

**排查步骤**：
1. 检查日志中是否有 `ReviewTrigger initialized`
2. 检查 `ReplayIntegration` 是否初始化成功
3. 检查事件总线是否正常

**解决方案**：
```bash
# 查看启动日志
python -m zulong.bootstrap 2>&1 | grep -i "review"

# 如果失败，重新初始化
# 修改 bootstrap.py 中的初始化顺序
```

### 问题 2：卡在模式选择阶段

**症状**：选择模式后无响应

**排查步骤**：
1. 检查 `_review_state_manager.get_stage()` 状态
2. 检查 `handle_mode_selection()` 是否被调用
3. 检查异步任务是否创建成功

**解决方案**：
```python
# 在 Gatekeeper 中添加日志
logger.info(f"当前阶段：{self._review_state_manager.get_stage()}")
```

### 问题 3：经验提取失败

**症状**：结束复盘后无经验生成

**排查步骤**：
1. 检查缓冲区是否有数据
2. 检查 L2 调用是否成功
3. 检查 JSON 解析是否失败

**解决方案**：
```python
# 检查缓冲区
buffer_manager = get_review_buffer_manager()
if buffer_manager.has_buffer():
    data = buffer_manager.get_buffer().export_for_analysis()
    print(f"缓冲区数据：{data}")
```

### 问题 4：状态不一致

**症状**：复盘模式标志为 True，但无法进行对话

**排查步骤**：
1. 检查 `state_manager.get_context('review_mode')`
2. 检查 `_review_state_manager.is_active()`
3. 检查两者是否一致

**解决方案**：
```python
# 统一使用 ReviewStateManager
if self._review_state_manager.is_active():
    # 处理复盘逻辑
```

## 📊 性能指标

### 正常性能指标

| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 启动时间 | < 2 秒 | 从输入到提示显示 |
| 模式切换 | < 1 秒 | 从选择到进入对话 |
| 经验提取 | < 10 秒 | 从结束到展示经验 |
| 对话响应 | < 3 秒 | L2 回复时间 |

### 性能优化技巧

1. **异步任务池**：避免频繁创建任务
2. **缓冲区优化**：使用 deque 限制长度
3. **批量状态同步**：减少 set_context 调用
4. **L2 调用优化**：使用线程池避免阻塞

## 📝 最佳实践

### 1. 正确使用复盘

✅ **推荐**：
- 在重要对话后使用快速复盘
- 在项目结束后使用深度复盘
- 定期复盘，积累经验

❌ **不推荐**：
- 频繁启动复盘（建议间隔>5 分钟）
- 在复盘模式下进行无关对话
- 忽略经验确认环节

### 2. 调试技巧

✅ **推荐**：
- 开启详细日志：`logging.basicConfig(level=logging.DEBUG)`
- 使用调试控制台：`/help` 查看所有命令
- 监控状态变化：订阅状态变更回调

❌ **不推荐**：
- 直接修改状态标志
- 跳过阶段检查
- 忽略异常处理

### 3. 扩展功能

✅ **推荐扩展**：
- 添加自定义触发条件
- 实现经验分类和标签
- 增加复盘历史记录
- 支持多人复盘

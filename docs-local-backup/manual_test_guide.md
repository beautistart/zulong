# 祖龙系统 - 手动测试流程

## 测试前准备

### 1. 确认系统状态
确保以下组件正在运行：

```bash
# 终端 1: vLLM 服务（WSL）
✅ 应该看到：vllm serve ... 运行中
✅ 验证：curl http://localhost:8000/v1/models

# 终端 2: 祖龙主系统
✅ 应该看到：ZULONG System Online
✅ 应该看到：Debug Console is ready

# 终端 3: OpenClaw Bridge
✅ 应该看到：Web 服务器已启动于 http://localhost:8080
✅ 应该看到：WebSocket 连接成功
```

### 2. 打开浏览器
访问：**http://localhost:8080**

应该看到一个聊天界面，有输入框可以输入文字。

---

## 测试流程

### 第一阶段：基础对话测试（5 分钟）

#### 测试 1.1: 简单问候
**操作步骤：**
1. 在浏览器输入框输入：`你好`
2. 按回车发送
3. 等待 5-10 秒

**预期结果：**
- ✅ AI 回复问候语（如"你好！有什么可以帮助你的吗？"）
- ✅ 响应时间在 10 秒以内

**失败情况：**
- ❌ 没有任何响应 → 系统事件路由有问题
- ❌ 响应超时（>30 秒） → 模型推理有问题

---

#### 测试 1.2: 多轮对话
**操作步骤：**
1. 输入：`你叫什么名字？`
2. 等待 AI 回复
3. 输入：`你今天心情怎么样？`
4. 等待 AI 回复

**预期结果：**
- ✅ AI 能正常回答两个问题
- ✅ 对话连贯，没有报错

---

### 第二阶段：短期记忆测试（10 分钟）

#### 测试 2.1: 用户信息记忆
**操作步骤：**

**第 1 轮：**
```
输入：我叫小明
等待：AI 回复（应该确认记住了你的名字）
```

**第 2 轮：**
```
输入：我今年 25 岁
等待：AI 回复（应该确认记住了你的年龄）
```

**第 3 轮：**
```
输入：我住在北京
等待：AI 回复（应该确认记住了你的地址）
```

**第 4 轮（关键测试）：**
```
输入：你还记得我叫什么吗？
等待：AI 回复
```

**预期结果：**
- ✅ AI 回答："你叫小明" 或类似内容
- ✅ 证明短期记忆功能正常

**失败情况：**
- ❌ AI 说"不记得"或"不知道" → 记忆模块有问题
- ❌ AI 没有反应 → 事件路由有问题

---

#### 测试 2.2: 记忆更新
**操作步骤：**

接上一个测试，继续：

**第 5 轮：**
```
输入：我改名字了，现在叫小红
等待：AI 回复
```

**第 6 轮：**
```
输入：那我现在叫什么？
等待：AI 回复
```

**预期结果：**
- ✅ AI 回答："你现在叫小红"
- ✅ 证明记忆可以更新

---

### 第三阶段：记忆持久化测试（5 分钟）

#### 测试 3.1: 跨会话记忆
**操作步骤：**

1. 刷新浏览器页面（F5 或 Ctrl+R）
2. 等待页面重新加载
3. 输入：`你还记得我的名字吗？`

**预期结果：**
- ✅ AI 回答："你叫小红"（更新后的名字）
- ✅ 证明记忆已持久化

**失败情况：**
- ❌ AI 说"不记得" → 持久化存储有问题
- ❌ 页面刷新后无法连接 → WebSocket 重连有问题

---

### 第四阶段：工具调用测试（10 分钟）

#### 测试 4.1: 工具调用意图识别
**操作步骤：**
```
输入：帮我搜索一下今天的天气
等待：15-30 秒
```

**预期结果（两种情况都算通过）：**
- ✅ AI 调用搜索工具并返回天气信息
- ✅ AI 回复："我目前无法访问外部 API，但可以..."（说明识别了工具调用意图）

**关键验证点：**
- AI 是否理解"搜索"是一个工具调用请求
- 是否有合理的响应（即使无法真正调用）

---

#### 测试 4.2: 复杂任务拆解
**操作步骤：**
```
输入：帮我创建一个 Python 项目，需要实现用户登录、数据管理、报表生成三个功能
等待：30-60 秒
```

**预期结果：**
- ✅ AI 尝试拆解任务（如"我们可以分步骤进行..."）
- ✅ AI 提到三个功能模块
- ✅ 有合理的响应结构

**失败情况：**
- ❌ 完全没有响应 → 任务调度有问题
- ❌ 响应混乱，没有逻辑 → L2 推理有问题

---

### 第五阶段：技能模块测试（10 分钟）

#### 测试 5.1: 技能包安装（如果有技能包）

```

```

**预期结果：**
- AI 列出可用的技能包或说明当前状态

---

#### 测试 5.2: 技能执行
**根据 AI 回复的技能列表，选择一个执行：**

例如，如果有计算器技能：
```
输入：帮我计算 123 * 456
```

如果有翻译技能：
```
输入：把"你好"翻译成英文
```

**预期结果：**
- ✅ 技能正确执行
- ✅ 返回合理结果

---

### 第六阶段：压力测试（可选，5 分钟）

## 常见问题排查

### 问题 1: 浏览器中没有任何响应
**可能原因：**
1. OpenClaw Bridge 没有运行
2. WebSocket 连接失败
3. 事件路由有问题

**排查步骤：**
```bash
# 1. 检查 OpenClaw Bridge 终端
# 应该看到：WebSocket 连接成功

# 2. 检查祖龙主系统终端
# 应该看到：ZULONG System Online

# 3. 查看是否有错误日志
# 查找 "ERROR" 或 "Exception"
```

---

### 问题 2: AI 响应超时（>30 秒）
**可能原因：**
1. vLLM 服务负载过高
2. 模型推理太慢
3. 系统资源不足

**排查步骤：**
```bash
# 1. 检查 vLLM 服务
curl http://localhost:8000/v1/models

# 2. 检查系统资源
# Windows: 任务管理器
# 查看 GPU 和 CPU 使用率

# 3. 重启 vLLM 服务（WSL 终端）
# Ctrl+C 停止，然后重新启动
```

---

### 问题 3: 记忆功能不工作
**可能原因：**
1. SharedMemoryPool 没有正确初始化
2. 记忆存储失败
3. 记忆检索逻辑有问题

**排查步骤：**
```bash
# 1. 查看祖龙主系统启动日志
# 应该看到：SharedMemoryPool 初始化完成

# 2. 检查记忆存储文件
# 路径：data/memory/ 或类似目录

# 3. 查看是否有记忆相关的错误日志
```

---

### 问题 4: 刷新页面后无法连接
**可能原因：**
1. WebSocket 没有自动重连
2. 会话状态丢失

**排查步骤：**
```bash
# 1. 刷新浏览器（F5）
# 2. 打开浏览器开发者工具（F12）
# 3. 查看 Console 标签，是否有 WebSocket 错误
# 4. 查看 Network 标签，WebSocket 连接状态
```

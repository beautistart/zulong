## 二、优势 (基于代码实证)

### 1. 统一异构图作为记忆底座 -- 架构级创新

`memory_graph.py` 使用单一 NetworkX DiGraph 统一承载 **9种节点类型**和 **7种边类型**：

- **节点类型**: TASK / DIALOGUE / KNOWLEDGE / EXPERIENCE / EPISODE / FILE / CONCEPT / PERSON / DOCUMENT
- **边类型**: HIERARCHY / DEPENDENCY / REFERENCE / TEMPORAL / SEMANTIC / CAUSAL / ASSOCIATION

市面上绝大多数项目 (Mem0, Zep, Letta) 的记忆子系统是独立平行的，而祖龙让所有记忆类型在同一张图上可以互相引用和传播。一条对话可以通过 REFERENCE 边连接到任务节点，再通过 HIERARCHY 边触及子任务，再通过 SEMANTIC 边关联到知识实体 -- 这种跨类型的上下文发现是市面项目做不到的。

### 2. 完整的认知计算三件套

#### BFS 扩散激活 (`memory_graph.py:1052-1126`)
```python
# 双向传播（出边 + 入边），共激活边记录供赫布学习消费
propagated = act * edge_weight * decay  # 衰减因子默认 0.5/跳
```
- 双向传播 (`out_edges` + `in_edges`)
- 同时记录共激活边对供赫布学习消费
- 最大深度3跳，最小激活阈值0.01

#### 赫布增强 (`memory_graph.py:1132-1205`)
```python
new_w = old_w + eta * (1.0 - old_w)  # eta=0.1, 渐近趋向1.0永不溢出
```
- 共激活计数阈值=3 才创建 ASSOCIATION 边
- 单节点 ASSOCIATION 出度上限10
- coactivation_counter 总容量上限5000
- 防爆保护完备

#### 艾宾浩斯衰减 (`memory_graph.py:1211-1305`)
```python
decayed = weight * exp(-elapsed_hours * ln(2) / half_life)
```
- 6级重要度对应不同半衰期：
  - TRIVIAL: 6h
  - NORMAL: 24h
  - IMPORTANT: 168h (7天)
  - FACT: 360h (15天)
  - IDENTITY: 720h (30天)
  - MUST_REMEMBER: 无穷大
- 边的半衰期取两端节点中更高重要度的值
- review_threshold=0.15 的濒危边交给 LLM 审查而非直接删除

**市面对比**: GAM 有 BFS 扩散但没有赫布学习和衰减；OpenCog Hyperon 有 ECAN 注意力分配但运行在 MeTTa 虚拟机上无法与 LLM 直接对接。祖龙是唯一一个将三者工程集成并实机验证的项目。

### 3. 热/冷双路检索

`retrieve_context()` (`memory_graph.py:1701-1739`)

**热路径** (`_retrieve_hot`, lines 1761-1891):
- 阶段0: 当前会话注入（session_id 匹配，分数 2.0）
- 阶段1: 元记忆查询检测（"记住/记得/回忆" 等关键词 → 注入 IDENTITY/FACT 节点）
- 阶段2: bigram 关键词匹配 + importance boost + session boost + 焦点子树 boost
- 阶段3: BFS 1跳扩散，焦点路径末尾3个节点作为额外种子

**冷路径** (`_retrieve_cold`, lines 1893-1942):
- FAISS 向量检索排除热节点
- 命中后 BFS 下钻获取子节点详情
- importance boost + 焦点子树 boost

两路通过 `asyncio.gather()` 并行执行，天然互斥无重复。

### 4. 工业级安全工程

| 安全机制 | 位置 | 说明 |
|----------|------|------|
| FC循环三层保护 | `inference_engine.py:1121-1143` | soft_limit(50) → hard_limit(100) → interrupt_flag |
| 5信号死循环检测 | `circuit_breaker.py` | 重复/模式循环/信息增益/上下文压力/墙钟时间 |
| 原子持久化 | `memory_graph.py` save() | temp→rename + .bak 崩溃恢复 |
| 赫布学习防爆 | `memory_graph.py:1163-1205` | 出度上限/counter容量/渐近公式 |
| 备用模型降级链 | `inference_engine.py:1168-1179` | 主模型→备用模型→fallback固定回复 |
| 结构性边永不修剪 | `memory_graph.py:73` | HIERARCHY/DEPENDENCY/TEMPORAL 豁免衰减 |

### 5. 注意力窗口 -- 自主思维层导航

`attention_window.py` 实现 3 种注意力模式：

| 模式 | 当前节点权重 | 无关内容权重 | 适用场景 |
|------|-------------|-------------|----------|
| GLOBAL | x1.5 (overview) | 正常 | 全局规划、查看进度 |
| FOCUS | x3.0 | 正常 | 聚焦某节点细节 |
| SINGLE_CHAIN | x5.0 | x0.2 | 深度推理、排除干扰 |

- **工具驱动状态机切换**：deeper → GLOBAL→FOCUS→SINGLE_CHAIN；broader 反向
- **`apply_window()` 7步算法**：评分→分离pinned→预算检查→按工具调用分组→排序→贪心选择→生成淘汰摘要
- **jump 模式**：根据目标节点深度自动选择注意力模式

市面上唯一一个让 LLM Agent 自主控制思维聚焦层级的实现。

### 6. 完整的任务中断-恢复链

5模块闭环：

```
RecoveryNotifier (启动扫描)
    → TaskSuspensionManager (状态序列化: 对话历史+CB状态+TaskGraph+迭代计数)
        → TaskStateManager (冻结/恢复桥接MemoryGraph)
            → SnapshotManager (LRU淘汰, max 5快照, 512MB内存上限)
                → DialogueAdapter.assign_session_by_similarity() (embedding相似度0.55阈值自动归入会话)
```

## 三、不足与风险

### 1. 图规模可伸缩性未验证 [风险: 中]

MemoryGraph 使用 Python 内存中的 NetworkX DiGraph。以下操作在图规模增长时延迟未知：
- `compute_activations()` 的 BFS 遍历 (每跳遍历 out_edges + in_edges)
- `decay_and_prune()` 的全图扫描 (遍历全部边)
- `_retrieve_hot()` 的全节点线性扫描 (`for nid, node in self._nodes.items()`)

当前单用户场景下图不会太大，商用化（多用户长期使用）需要引入分片或 lazy loading。

### 2. 对 LLM 能力的强依赖 [风险: 高]

系统效果高度依赖 LLM 的 Function Calling 能力：
- 两阶段意图分类 (`_classify_intent`) 依赖 LLM 正确调用 `start_session` 工具
- 思维层导航依赖 LLM 主动调用 `navigate_attention` 工具
- 任务分解依赖 LLM 正确调用规划工具

RTX 3060 6GB 上本地模型的 FC 准确率直接限制系统上限。这是架构级天花板。

### 3. 缺乏量化评测基线 [风险: 高]

代码有丰富的日志和统计计数器，但缺少：
- 标准化记忆检索准确率测试 (precision/recall)
- 注意力窗口裁剪前后任务完成率对比
- 赫布学习启用/不启用的 A/B 测试
- 与 Mem0/Zep 等项目的端到端对比

"跑通"和"可量化验证优于竞品"之间有巨大差距。

### 4. 单线程事件处理 [风险: 低]

AttentionController 使用 `queue.PriorityQueue` 做事件排队，`on_l2_idle()` 每次最多处理3个事件。当高优先级事件密集到来时存在积压风险。当前单用户场景下不太可能触发。

### 5. 重要度检测依赖正则规则 [风险: 中]

`DialogueAdapter._detect_importance()` 使用正则匹配关键词判断重要度：
- "叫我/我叫/名字是" → IDENTITY
- "记住/记下/别忘了" → MUST_REMEMBER

脆弱之处：英文内容几乎不命中；隐含重要信息（如"我有花生过敏"）可能被标记为 NORMAL。

## 四、价值点

### 价值点 1: 可运行的认知架构参考实现
SOAR/ACT-R 是 Lisp/Java 时代产物，OpenCog Hyperon 无稳定 LLM 集成。祖龙是目前唯一基于现代 LLM + FC + 图记忆 + 注意力控制的完整可运行系统。

### 价值点 2: "记忆是活的"范式验证
BFS扩散 + 赫布增强 + 艾宾浩斯衰减的组合验证了：AI记忆可以像生物神经网络一样"活"起来 -- 常用连接越来越强，不用连接逐渐消退，频繁共激活的概念自动产生关联。

### 价值点 3: 注意力窗口管理的工程范式
`AttentionWindowManager` 的解决方案（三模式 x 工具驱动状态机 x token预算贪心分配 x 淘汰摘要注入）脱离祖龙也有独立价值。

### 价值点 4: 任务中断-恢复的完整工程方案
5模块恢复链（Recovery Notifier → Suspension Manager → State Manager → Snapshot Manager → Session Binding）是目前可见的最完整的跨会话任务恢复实现。

## 五、竞争力对比矩阵

| 维度 | 祖龙 | Mem0/Zep | Letta/MemGPT | OpenCog Hyperon | GAM |
|------|------|----------|-------------|-----------------|-----|
| 记忆统一图 | 9种节点+7种边, 单图 | KV存储, 无图 | 分层存储, 无图 | MeTTa Atomspace | 单类型图 |
| BFS扩散激活 | 双向BFS+共激活记录 | 无 | 无 | ECAN(理论) | 单向BFS |
| 赫布学习 | 渐近公式+防爆限制 | 无 | 无 | 设计中 | 无 |
| 衰减修剪 | 艾宾浩斯+LLM审查 | 无 | 无 | STI递减 | 无 |
| 注意力模式 | 3模式+工具驱动切换 | 无 | 无 | ECAN注意力 | 无 |
| 任务中断恢复 | 5模块完整链 | 无 | 无 | 无 | 无 |
| 热/冷双路检索 | 并行+互斥+焦点boost | 向量检索 | 向量+摘要 | 模式匹配 | 向量检索 |
| 实际可运行 | RTX 3060验证 | 云API | 云API | 部分模块 | 需GPU |

### 核心竞争差异化

1. **深度集成**: 别的项目是"记忆模块"，祖龙是"认知系统"。从记忆存储到检索到激活到衰减到注意力控制到任务管理，是一条完整的认知链路。
2. **生物启发工程化**: 赫布学习、艾宾浩斯衰减、BFS扩散激活不是论文中的概念，而是带有防爆安全机制的工程代码。
3. **自主注意力**: 市面上没有第二个项目让 LLM Agent 通过 FC 工具自主控制思维聚焦层级。

## 六、总结判断

祖龙系统的代码质量远超"个人项目"的平均水平，工程化程度（安全保护、原子持久化、降级链、防爆限制）接近生产级。其核心竞争力在于**将认知科学概念工程化为可运行系统**这件事本身 -- 这在当前 AI Agent 领域是稀缺的。

主要短板不在代码，而在**验证层**：需要量化评测证明"赫布学习确实提升了检索质量"、"注意力窗口确实提高了长任务完成率"。有了这些数据，祖龙从"有趣的项目"升级为"有说服力的系统"的路径就打通了。

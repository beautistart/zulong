# 祖龙(ZULONG)开源方案详细设计

> **版本**: v1.0
> **日期**: 2026-05-12
> **目标**: 保护核心创新 + 扩大影响力 + 清晰的商业化路径

---

## 一、总体策略

### 核心原则

```
1. 核心创新保护 → AGPL-3.0
   └─ MemoryGraph、CircuitBreaker、InferenceEngine

2. 接口开放友好 → MIT
   └─ 前端、API定义、工具接口

3. 文档开放传播 → CC BY-NC-SA 4.0
   └─ 允许传播，禁止商业使用

4. 配置和示例 → MIT
   └─ 降低使用门槛
```

## 十、总结

### 许可证分布

| 许可证 | 范围 | 目的 | 代码行数 |
|--------|------|------|---------|
| **AGPL-3.0** | 核心创新 | 保护核心竞争力 | ~25,000 |
| **MIT** | 接口前端 | 鼓励集成使用 | ~20,000 |
| **CC BY-NC-SA 4.0** | 文档 | 允许传播，禁止商业 | ~2000页 |

### 保护效果

```
AGPL-3.0 保护：
├─ 企业SaaS使用 → 必须开源
├─ 嵌入闭源产品 → 必须购买授权
└─ 提供商业谈判筹码

MIT 开放：
├─ 开发者可自由使用前端
├─ 鼓励集成到其他项目
└─ 降低使用门槛

CC BY-NC-SA 4.0 文档：
├─ 允许传播分享
├─ 必须署名
└─ 禁止商业使用文档
```

### 这是最优方案，因为：

1. **保护核心** - MemoryGraph、CircuitBreaker、InferenceEngine是核心竞争力
2. **开放接口** - 前端和API层开放，鼓励开发者使用
3. **清晰边界** - 哪些AGPL、哪些MIT，非常明确
4. **商业路径** - 企业想用核心，要么开源，要么付费
5. **法律威慑** - 大厂看到AGPL会谨慎评估

## 🔧 阈值配置逻辑

### Layer 3 阈值配置

#### intent_threshold: 0.25

**配置理由**:
1. 平均分数为 0.193，设置 0.25 可以过滤掉大部分静止状态
2. SITTING 平均 0.259，设置 0.25 可以稳定检测到坐姿
3. 38.3% 的帧超过 0.2，说明 0.25 是合理的检测门槛

**预期效果**:
- 静止状态：< 0.25 (不触发)
- 小幅动作：0.25-0.35 (触发 SILENT_WATCH)
- 明显动作：> 0.35 (触发 INTERACT_REQUEST)

#### interact_threshold: 0.35

**配置理由**:
1. 最高分数 0.573，设置 0.35 确保能触发 Layer 4
2. 16.7% 的帧超过 0.3，说明 0.35 不会太频繁
3. 如果设置 > 0.4，只有 3.3% 的帧能触发，太少了

**预期效果**:
- 当 L3 分数 >= 0.35 时，触发 Layer 4 鹰眼模式
- 在 SITTING、部分 STANDING 和 LYING 时能触发
- 不会过于频繁（约 10-15% 的时间）

### Layer 4 阈值配置

#### gesture_conf_threshold: 0.25

**配置理由**:
1. MediaPipe 诊断工具使用 0.3，工作良好
2. 设置 0.25 略低于诊断工具，提高灵敏度
3. 避免过低阈值 (0.15) 导致的误识别

**预期效果**:
- 清晰手势：置信度 > 0.5 (稳定识别)
- 一般手势：置信度 0.3-0.5 (可以识别)
- 模糊手势：置信度 < 0.25 (过滤掉)

#### eagle_eye_cooldown: 0.3

**配置理由**:
1. 0.3 秒 = 300ms，足够快速响应
2. 避免频繁触发消耗资源
3. 与 L3 的帧率 (12.6 FPS) 匹配

## 🎯 OK 手势识别问题分析

### 问题现象
在 `test_layer4_manual.py` 中，OK 手势经常识别失败

### 可能原因

#### 1. 手势角度问题
**分析**: MediaPipe 的 OK 手势识别对角度的要求较高
- 需要拇指和食指形成完整的圆圈
- 手掌角度需要面向摄像头

**解决**:
- 确保手掌面向摄像头
- 拇指和食指完全接触形成圆圈
- 其他三指伸直

#### 2. 触发时机问题
**分析**: Layer 4 需要 Layer 3 先触发
- 如果做 OK 手势时 L3 分数 < 0.35，Layer 4 不会启动
- OK 手势通常动作较小，L3 分数可能不高

**解决**:
- 做 OK 手势前先挥手或身体前倾，提高 L3 分数
- 或者降低 interact_threshold 到 0.3

#### 3. 置信度阈值问题
**分析**: OK 手势的置信度可能偏低
- MediaPipe 对 OK 手势的识别置信度可能不如 V 字和点赞高

**解决**:
- 已在配置中设置 gesture_conf_threshold = 0.25
- 如果仍然不行，可以降低到 0.2

### 测试建议

**步骤 1**: 先触发 Layer 3
```
1. 挥手 2-3 秒 (提高 L3 分数)
2. 身体前倾 (保持 L3 分数)
3. 然后比出 OK 手势
```

**步骤 2**: 观察日志
```
查看 Layer 3 分数是否 >= 0.35
查看 Layer 4 是否触发
查看 OK 手势的置信度
```

**步骤 3**: 调整手势
```
- 确保 OK 手势标准
- 距离摄像头 50-80cm
- 光线充足
- 手掌完全面向摄像头
```

## 📋 配置总览

### 当前配置

```python
# Layer 3 配置
'intent_threshold': 0.25,      # 意图检测阈值
'interact_threshold': 0.35,    # 交互触发阈值

# Layer 4 配置
'gesture_conf_threshold': 0.25,  # 手势置信度阈值
'eagle_eye_cooldown': 0.3,       # 鹰眼冷却时间

# MediaPipe 配置
min_hand_detection_confidence: 0.3   # 手部检测阈值
min_hand_presence_confidence: 0.3    # 手部存在阈值
min_tracking_confidence: 0.3         # 手部跟踪阈值
```

### 配置特点

1. **数据驱动**: 基于 95.9 秒实测数据调整
2. **平衡性**: 既保证灵敏度，又避免误触发
3. **渐进式**: L3 和 L4 阈值形成梯度

### 预期表现

**Layer 3**:
- 静止状态：不触发 (分数 < 0.25)
- 小幅动作：触发 SILENT_WATCH (0.25-0.35)
- 明显动作：触发 INTERACT_REQUEST (> 0.35)

**Layer 4**:
- 当 L3 >= 0.35 时触发
- 手势置信度 >= 0.25 时识别
- 每 0.3 秒可以触发一次

## 🔬 进一步优化方向

### 如果 Layer 4 触发太频繁

**方案**: 提高 interact_threshold
```python
'interact_threshold': 0.35 -> 0.4  # 减少触发
```

### 如果 Layer 4 触发太少

**方案**: 降低 interact_threshold
```python
'interact_threshold': 0.35 -> 0.3  # 增加触发
```

### 如果手势识别不准确

**方案 1**: 提高 gesture_conf_threshold
```python
'gesture_conf_threshold': 0.25 -> 0.3  # 减少误识别
```

**方案 2**: 降低 threshold 提高灵敏度
```python
'gesture_conf_threshold': 0.25 -> 0.2  # 提高灵敏度
```

### 如果 OK 手势仍然识别不了

**方案 1**: 检查手势标准度
- 确保拇指和食指形成完整圆圈
- 其他三指伸直

**方案 2**: 降低 OK 手势的识别门槛
```python
# 在 mediapipe_gesture_recognizer.py 中
# 添加 OK 手势的特殊处理
if gesture_name == "OK_Gesture":
    confidence *= 1.2  # 提高 OK 手势的置信度
```

**方案 3**: 使用备用方案
```python
# 如果 MediaPipe 识别不到 OK，使用 EfficientNet
if gesture_name is None:
    gesture_name, conf = efficientnet_classify(hand_roi)
```

### 关键指标

运行测试时，关注以下指标：

1. **Layer 3 触发频率**

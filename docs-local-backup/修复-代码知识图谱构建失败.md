# 修复-代码知识图谱构建失败问题

## 问题现象

日志显示：
```
[INFO] zulong.code.graph_builder: [CodeGraph] 扫描到 500 个文件 (python, typescript, javascript)
[ERROR] zulong.tools.code_tools: [IndexProjectTool] 后台索引异常: 不支持的语言: javascript
```

代码知识图谱构建失败，无法索引JavaScript/TypeScript文件。

## 问题根因

### 1. tree-sitter语言包未安装

`zulong/code/ast_parser.py`第33-46行代码支持javascript和typescript：

```python
elif lang == "javascript":
    try:
        import tree_sitter_javascript as tsjs
        language = Language(tsjs.language())
    except ImportError:
        logger.warning(f"[ASTParser] tree-sitter {lang} 语言包未安装，跳过")
        return None
```

但实际环境中缺少对应的Python包：
- `tree-sitter-javascript`
- `tree-sitter-typescript`

### 2. 异常处理不完善

`zulong/tools/code_tools.py`第1122-1123行：

```python
except Exception as bg_err:
    logger.error(f"[IndexProjectTool] 后台索引异常: {bg_err}")
```

后台线程遇到异常后只记录日志，没有：
- 广播失败通知给前端
- 提供降级方案（如跳过不支持的语言）
- 返回部分成功的结果

## 修复方案

### 方案1：安装缺失的tree-sitter语言包（推荐）

```bash
pip install tree-sitter-javascript tree-sitter-typescript
```

**优点**：完整支持所有语言
**缺点**：需要额外的Python包依赖

### 方案2：代码层面自动跳过不支持的语言

修改`zulong/code/graph_builder.py`的`build`方法，在扫描文件时检查语言包是否可用，不可用时跳过并记录警告。

**优点**：健壮性更好，不会因部分语言不支持而完全失败
**缺点**：部分文件无法索引

### 方案3：只索引已安装语言包的语言

修改`IndexProjectTool`的默认语言列表，从`["python"]`改为只包含已安装的语言。

## 推荐修复步骤

### ✅ 已实施修复（代码层面）

**修改文件**：`zulong/tools/code_tools.py`

1. **添加语言包可用性检查**（第854-880行）
   ```python
   # 检查语言包可用性，过滤掉不支持的语言
   available_languages = []
   for lang in languages:
       parser = ASTParser(lang)
       if parser.available:
           available_languages.append(lang)
       else:
           logger.warning(
               f"[IndexProjectTool] 跳过语言 {lang}：tree-sitter 语言包未安装"
           )
   ```

2. **使用过滤后的语言列表**（第883行）
   ```python
   code_graph = bg_builder.build(
       resolved_root,
       languages=available_languages,  # 使用过滤后的语言列表
       max_files=max_files,
   )
   ```

3. **增强异常处理**（第1147-1169行）
   - 广播`CRG_INDEX_FAILED`事件到前端
   - 提供详细的错误信息和项目名称

### 可选：安装完整的tree-sitter语言包

如需支持JavaScript/TypeScript索引：
```bash
pip install tree-sitter-javascript tree-sitter-typescript
```

# 祖龙系统技能包复杂任务测试方案

## 📋 测试目标

验证祖龙系统在面对复杂、多步骤任务时，能否正确调用技能包（Skill Packs）和工具（Tools），包括：
- 任务分解能力
- 工具选择能力
- 多步骤执行能力
- 信息整合能力
- 错误处理能力

## 🎯 测试任务设计

### 任务 1：市场研究与竞争分析（综合型）

**任务描述**：
```
请帮我分析当前 AI 助手市场的竞争格局，并完成以下工作：
1. 搜索 2025-2026 年主流 AI 助手产品（至少 5 个）
2. 对比它们的核心功能、定价策略、目标用户
3. 分析每个产品的优势和劣势
4. 找出市场空白点和机会
5. 为新产品提出 3 个差异化建议
```

**预期技能调用**：
- ✅ `WebSearchTool` / `OpenClawSearchTool` - 搜索市场信息
- ✅ `OpenClawPlugin` - 抓取产品网页详情
- ✅ `AutoGPTPlanner` - 任务分解和规划
- ✅ `OpenManusReasoner` - 分析和推理
- ✅ `ClineCoder` - 可能用于数据整理

**验证点**：
- [ ] 是否调用搜索工具获取最新信息
- [ ] 是否进行多轮搜索（不同关键词）
- [ ] 是否抓取网页获取详细信息
- [ ] 是否有结构化的分析框架
- [ ] 是否提供可执行的建议

## 📊 评估标准

### 技能调用维度

| 维度 | 优秀 (5 分) | 良好 (3 分) | 差 (1 分) |
|------|-------------|-------------|-----------|
| **工具选择** | 准确选择最合适的工具 | 选择了可用但非最优的工具 | 工具选择错误 |
| **调用时机** | 在恰当的时机调用工具 | 调用时机基本合理 | 调用时机混乱 |
| **调用频率** | 频率适中，不过度调用 | 调用次数合理 | 过度调用或调用不足 |
| **参数传递** | 参数准确完整 | 参数基本正确 | 参数错误或缺失 |
| **结果处理** | 正确处理并整合结果 | 能使用结果但有遗漏 | 无法处理结果 |

### 任务完成维度

| 维度 | 优秀 (5 分) | 良好 (3 分) | 差 (1 分) |
|------|-------------|-------------|-----------|
| **完整性** | 完成所有子任务 | 完成大部分子任务 | 只完成少量任务 |
| **准确性** | 信息准确可靠 | 大部分信息准确 | 信息错误较多 |
| **深度** | 分析深入有洞见 | 分析基本到位 | 分析肤浅 |
| **结构化** | 结构清晰逻辑强 | 结构基本完整 | 结构混乱 |
| **可操作性** | 建议具体可执行 | 建议基本可行 | 建议空洞 |

## 🚀 快速测试脚本

创建测试脚本 `test_skill_packs_complex.py`：

```python
import asyncio
import websockets
import json
import logging

# 配置日志
logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

# 测试任务列表
TEST_TASKS = [
    # 任务 1
    """请帮我分析当前 AI 助手市场的竞争格局，并完成以下工作：
    1. 搜索 2025-2026 年主流 AI 助手产品（至少 5 个）
    2. 对比它们的核心功能、定价策略、目标用户
    3. 分析每个产品的优势和劣势
    4. 找出市场空白点和机会
    5. 为新产品提出 3 个差异化建议""",
    
    # 任务 2
    """我需要了解 2026 年人工智能领域的最新技术趋势，请：
    1. 搜索 2025-2026 年 AI 领域的突破性技术（至少 3 个方向）
    2. 查找每个技术方向的代表性论文或产品
    3. 分析这些技术的发展阶段（萌芽/成长/成熟）
    4. 评估商业应用前景
    5. 生成一份结构化报告（包含摘要、正文、结论）""",
    
    # 添加更多任务...
]

async def test_task(task_id: int, task: str):
    """测试单个任务"""
    uri = "ws://localhost:5555"
    
    try:
        async with websockets.connect(uri) as websocket:
            logger.info(f"\n{'='*60}")
            logger.info(f"开始测试任务 {task_id}")
            logger.info(f"{'='*60}")
            
            # 发送任务
            logger.info(f"发送任务：{task[:100]}...")
            await websocket.send(task)
            
            # 接收响应
            response_count = 0
            async for message in websocket:
                response_count += 1
                logger.info(f"收到响应 #{response_count}: {message[:200]}...")
                
                # 简单判断是否完成
                if "完成" in message or "总结" in message:
                    logger.info("✅ 任务完成")
                    break
                
                # 超时保护（最多接收 10 条消息）
                if response_count >= 10:
                    logger.warning("⚠️ 达到最大响应数量，可能未完成")
                    break
                    
    except Exception as e:
        logger.error(f"❌ 测试失败：{e}")

async def main():
    """主测试函数"""
    logger.info("开始祖龙系统技能包复杂任务测试")
    
    for i, task in enumerate(TEST_TASKS, 1):
        await test_task(i, task)
        # 任务间间隔
        await asyncio.sleep(5)
    
    logger.info("\n所有测试完成！")

if __name__ == "__main__":
    asyncio.run(main())
```

## 💡 使用建议

1. **逐步测试**：从简单任务开始，逐步增加复杂度
2. **详细记录**：记录每次测试的日志和输出
3. **对比分析**：对比不同任务的表现，找出规律
4. **持续优化**：根据测试结果优化技能包配置
5. **回归测试**：定期执行测试，确保功能稳定

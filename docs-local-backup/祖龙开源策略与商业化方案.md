## 1. 技术差异化评估

### 1.1 真正领先的部分

#### MemoryGraph 异构记忆图谱

祖龙最有价值的子系统，在开源 agent 框架中具备明确的技术领先性：

| 能力 | 祖龙 | LangGraph | AutoGPT | MemGPT/Letta | CrewAI |
|------|------|-----------|---------|--------------|--------|
| 持久化异构图 | NetworkX + JSON | 内存 DAG | 基于文件 | 无图结构 | 无 |
| 赫布学习 | 共激活边自动增强 | 无 | 无 | 无 | 无 |
| 艾宾浩斯衰减 | exp 衰减 + 突触修剪 | 无 | 基于年龄 | 无 | 无 |
| FAISS 双路径检索 | 热路径 BFS + 冷路径向量 | 无 | 单路向量 | 单路 | 无 |
| 语义边自动发现 | 余弦相似度自动创建 | 无 | 无 | 无 | 无 |

**市场定位**: "AI Agent 的异构记忆图谱引擎"。长期记忆是 AI agent 领域公认的核心痛点，OpenAI ChatGPT Memory、Google Project Astra 都在做，但开源界的实现远不如祖龙完整。

**核心技术细节**:
- **赫布学习 (Hebbian Learning)**: 共激活的边自动增强权重，实现"经验累积"效应
- **艾宾浩斯衰减**: `score *= exp(-decay * age_days)`，0天100% → 7天70% → 30天22% → 60天5%淘汰
- **BFS 扩散激活**: 从种子节点触发 BFS，逐级扩散激活值，层级递减权重
- **双路径检索**: 热路径（邻域节点 + 焦点直连）+ 冷路径（FAISS 向量索引 + 余弦相似度）
- **语义边自动发现**: 后台持续计算节点间余弦相似度，超过阈值自动创建 SEMANTIC 边

#### Circuit Breaker 5 信号死循环检测

| 项目 | 检测方式 |
|------|----------|
| 祖龙 | 5 信号综合（重复调用、模式循环、信息增益递减、上下文压力、时间超限） |
| LangChain | max_iterations 硬限制 |
| CrewAI | max_iterations 硬限制 |
| OpenDevin | 时间/步数限制 |

**独特性**: 信息增益递减检测（Jaccard 相似度判断工具返回重叠率）在业界罕见。

#### 跨天级任务挂起/恢复

完整的 `SuspendableTaskState` 序列化（含对话历史、CB 状态、TaskGraph），支持天级别暂停与恢复。CrewAI、MetaGPT、AutoGPT 均不支持。

**适用场景**: 24 小时陪伴式机器人、超长程项目管理。

### 1.2 有亮点但不算领先的部分

- **两阶段 FC 意图分类**: 架构合理，但 LangGraph 的 DAG 路由、Semantic Kernel 的 Planner 做类似事情
- **注意力窗口三模式**: GLOBAL/FOCUS/SINGLE_CHAIN 思路好，但与 MemoryGraph 的 BFS 激活尚未打通
- **四层推理架构**: 设计精良，但 L0/L1 感知层仅在硬件机器人场景有意义

### 1.3 综合对标

| 维度 | 祖龙 vs LangGraph | 祖龙 vs CrewAI | 祖龙 vs MemGPT/Letta | 祖龙 vs MetaGPT |
|------|-------------------|----------------|----------------------|-----------------|
| 记忆系统 | 领先 | 领先 | 领先（异构融合） | 领先 |
| 中断恢复 | 领先 | 领先 | 持平 | 领先 |
| 代码成熟度 | 落后 | 落后 | 持平 | 落后 |
| 社区生态 | 落后 | 落后 | 落后 | 落后 |
| 多 Agent 协作 | 落后 | 落后 | 不适用 | 落后 |

**结论**: 记忆系统和死循环检测是核心竞争力，其余部分不具备差异化优势。

## 2. 开源策略选择

### 2.1 许可证对比

| 许可证 | 传播力 | 商业保护 | 代表项目 | 适合场景 |
|--------|--------|----------|----------|----------|
| Apache 2.0 | 最强 | 最弱 | K8s, TensorFlow, LangChain | 有执行速度优势的大团队 |
| AGPL-3.0 | 中等 | 较强 | MongoDB(早期), Grafana | 个人/小团队需要保护核心 |
| BSL | 较弱 | 最强 | MariaDB, Sentry, HashiCorp | 明确阻止竞争性使用 |
| 双重许可 | 中等 | 强 | MySQL, Qt | 有法务资源的团队 |

### 2.2 推荐方案: AGPL-3.0 + 商业例外

**选择理由**:

1. **团队规模约束**: 个人开发者没有大厂的执行速度优势。Apache 许可下大厂 fork 代码加几个工程师一周就能超越。AGPL 逼迫他们要么开源自己的产品，要么付费。
2. **AGPL 传染性即护城河**: 企业想在闭源产品中集成 zulong-memory，必须购买商业许可。这是自然的收入来源。
3. **LangChain 用户场景**: 用户想用记忆系统但不想让自己项目变成 AGPL，付费即可。自然的商业化路径。

**使用条件**:
- 个人/学术/内部使用: 免费
- 开源项目集成: 免费（AGPL 传染）
- 商业 SaaS / 闭源产品集成: 需购买商业许可

## 3. 分层开源架构

```
┌─────────────────────────────────────────────────┐
│  祖龙完整系统（闭源 / 付费授权）                  │
│  ├── L2 推理引擎 + 两阶段 FC                     │
│  ├── Circuit Breaker 5 信号                      │
│  ├── 任务挂起/恢复 + 注意力窗口                   │
│  ├── L1 感知层 + L0 设备层                        │
│  └── 完整集成 + 调优 + 部署方案                   │
├─────────────────────────────────────────────────┤
│  zulong-memory（AGPL-3.0 开源）                   │
│  ├── MemoryGraph 异构图结构核心                    │
│  ├── 赫布学习 / 艾宾浩斯衰减 / BFS 扩散           │
│  ├── FAISS 双路径检索                             │
│  ├── 语义边自动发现                               │
│  └── 基础 API + 文档 + 示例                       │
├─────────────────────────────────────────────────┤
│  zulong-circuit-breaker（Apache 2.0 开源）        │
│  └── 通用 agent 死循环检测（引流用）               │
└─────────────────────────────────────────────────┘
```

**为什么 Circuit Breaker 用 Apache 而 MemoryGraph 用 AGPL**:
- Circuit Breaker 功能相对简单，独立价值有限，用 Apache 宽松许可最大化传播，作为引流入口
- MemoryGraph 是核心竞争力，用 AGPL 保护商业利益

**祖龙主仓库改造**:
```python
# zulong/memory/memory_graph.py 改为依赖:
from zulong_memory import MemoryGraph, NodeType, EdgeType
# 祖龙特有的适配逻辑保留在 zulong 仓库内
```

## 4. 商业模式设计

### 4.1 短期收入（开源后 3-6 个月）

#### 商业许可费
- 企业想在闭源产品中集成 zulong-memory → 购买商业许可
- 定价参考: 小团队 $500/年，企业 $5000/年
- AGPL 传染性自然驱动

#### 付费插件 / 高级功能
- **开源版**: 基础图操作 + 检索
- **付费版**:
  - 分布式多节点 MemoryGraph（PostgreSQL/Redis 后端）
  - 高性能向量引擎（GPU FAISS / Milvus 集成）
  - 可视化管理面板
  - 企业级监控 + 审计日志

### 4.2 中期收入（6-12 个月）

#### 祖龙完整系统授权
- 面向机器人厂商 / 智能硬件公司
- "基于 zulong-memory 的完整机器人智能体方案"
- 包含推理引擎、任务管理、感知层的完整集成

#### 咨询与定制
- 帮企业基于 zulong-memory 构建自己的 agent 记忆系统

#### 云服务 (Memory-as-a-Service)

## 5. zulong-memory 开源库规划

### 5.1 目录结构

```
zulong-memory/
├── src/zulong_memory/
│   ├── __init__.py           # 公开 API
│   ├── graph.py              # 核心异构图结构（6 种节点, 7 种边）
│   ├── learning.py           # 赫布学习 + 艾宾浩斯衰减 + 突触修剪
│   ├── activation.py         # BFS 扩散激活
│   ├── retrieval.py          # 双路径检索（热路径 + FAISS 冷路径）
│   ├── discovery.py          # 语义边自动发现

│   └── types.py              # NodeType, EdgeType 等类型定义
├── examples/
│   ├── basic_usage.py        # 基础用法
│   ├── langchain_integration.py  # LangChain 集成示例
│   ├── crewai_integration.py     # CrewAI 集成示例
│   └── agent_memory.py       # 完整 agent 记忆示例
├── benchmarks/
│   ├── retrieval_benchmark.py    # 检索性能对比
│   └── vs_langchain_memory.py    # 与 LangChain Memory 对比
├── tests/
│   ├── test_graph.py
│   ├── test_learning.py
│   ├── test_retrieval.py
│   └── test_persistence.py

```

### 5.2 从 memory_graph.py 抽取的模块边界

| 源文件功能 | 目标模块 | 行数估计 |
|------------|----------|----------|
| GraphNode, NodeType, EdgeType 定义 | types.py | ~100 |
| 图结构 CRUD (add_node, add_edge, get_node) | graph.py | ~600 |
| hebbian_strengthen, decay_and_prune | learning.py | ~300 |
| compute_activations (BFS) | activation.py | ~200 |
| retrieve_context (热+冷双路径) | retrieval.py | ~400 |
| FAISS sidecar + 语义边自动创建 | discovery.py | ~300 |
| serialize/deserialize, JSON I/O | persistence.py | ~200 |

**总计约 2100 行核心代码**，一个人可控。

### 5.3 README 核心要素

README 决定项目 80% 的传播力，必须包含:

1. **一句话定位**: "AI Agent 的异构记忆图谱引擎 —— 让你的 Agent 真正拥有长期记忆"
2. **与竞品的对比表**: vs LangChain Memory / MemGPT / 原生向量检索
3. **10 行代码快速上手**:
```python
from zulong_memory import MemoryGraph, NodeType, EdgeType

mg = MemoryGraph()
mg.add_node("task_1", NodeType.TASK, label="写报告", content="完成季度报告")
mg.add_node("knowledge_1", NodeType.KNOWLEDGE, label="报告格式", content="...")
mg.add_edge("task_1", "knowledge_1", EdgeType.REFERENCE)

# 自动学习: 共激活的节点连接增强
mg.hebbian_strengthen("task_1", "knowledge_1")

# 检索: 热路径 + 向量双路径融合
results = mg.retrieve_context("如何写季度报告", top_k=5)
```
4. **效果 GIF / 可视化截图**
5. **2-3 个 benchmark 数据**: 检索准确率、学习收敛速度、衰减效果

## 附录: 风险提醒

**开源最大的风险不是被抄，而是没人用。**

AGPL 许可在保护利益的同时会降低传播速度。前期最重要的不是许可证选择，而是:

1. 代码质量和文档是否让人 5 分钟内就能跑通
2. 是否解决了 LangChain/LangGraph 用户的真实痛点
3. 是否有 2-3 个有说服力的 benchmark

先让它有用，再保护它。

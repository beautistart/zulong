# 祖龙系统技能包测试指南

## 📋 测试目标

验证祖龙系统在面对复杂任务时，能否正确调用技能包和工具。

## 🚀 快速开始

### 方法 1：使用自动化测试脚本（推荐）

```powershell
# 1. 确保系统已启动
# 启动祖龙系统
cd d:\AI\project\zulong_beta4
python zulong\bootstrap.py

# 启动 OpenClaw Bridge（新终端）
python openclaw_bridge\bootstrap.py

# 2. 运行测试脚本（新终端）
python test_skill_packs_complex.py
```

### 方法 2：手动测试（更灵活）

通过祖龙系统的调试控制台或 WebSocket 客户端，手动发送测试任务。

## 📝 测试任务列表

### 任务 1：简单搜索（入门级）
**目的**：验证基础搜索功能

```
请搜索 2026 年人工智能领域的最新动态，告诉我 3 个重要新闻。
```

**预期行为**：
- ✅ 调用 WebSearchTool 或 OpenClawSearchTool
- ✅ 返回 3 条新闻
- ✅ 包含新闻来源和时间

---

### 任务 2：信息对比（进阶级）
**目的**：验证信息整合和对比能力

```
我想了解当前主流的 AI 大模型（如 GPT-4、Claude、Gemini 等），请：
1. 搜索这些模型的基本信息
2. 对比它们的参数规模、训练数据、核心能力
3. 用表格形式展示对比结果
```

**预期行为**：
- ✅ 多次搜索获取不同模型信息
- ✅ 结构化对比（表格）
- ✅ 信息准确完整

---

### 任务 3：市场分析（复杂级）
**目的**：验证复杂任务分解和多工具协同

```
请帮我分析 AI 助手市场的竞争格局：
1. 搜索 2025-2026 年主流 AI 助手产品（至少 5 个）
2. 对比它们的核心功能、定价策略
3. 分析每个产品的优势和劣势
4. 找出市场机会点
5. 提出 3 个差异化建议
```

**预期行为**：
- ✅ 调用多种工具（搜索、抓取、分析、规划）
- ✅ 分步骤执行
- ✅ 提供深入分析和可执行建议

---

### 任务 4：技术方案设计（专家级）
**目的**：验证创造性思维和专业技术能力

```
我需要开发一个教育行业的 AI 应用，请帮我：
1. 调研教育行业 AI 应用的现状和痛点
2. 分析目标用户（学生、教师）的需求
3. 设计 5 个核心功能
4. 规划技术架构（前端、后端、AI 模型）
5. 估算开发成本和周期
```

**预期行为**：
- ✅ 深入调研
- ✅ 需求分析准确
- ✅ 技术方案合理
- ✅ 成本估算有依据

---

### 任务 5：实时新闻分析（时效性）
**目的**：验证实时信息处理能力

```
请搜索最近 7 天内关于"人工智能监管"的新闻，然后：
1. 整理主要事件（时间、地点、参与方）
2. 分析各国监管政策差异
3. 评估对 AI 行业的影响
4. 预测未来监管趋势
5. 为企业提出合规建议
```

**预期行为**：
- ✅ 新闻时效性强（7 天内）
- ✅ 多来源验证
- ✅ 分析有深度
- ✅ 建议可操作

## 🔍 验证方法

### 1. 查看日志

**L2 推理引擎日志**：
```powershell
Get-Content "d:\AI\project\zulong_beta4\zulong\l2\logs\l2_*.log" -Tail 100 | Select-String -Pattern "工具 | 搜索 | 调用"
```

**OpenClaw Bridge 日志**：
```powershell
Get-Content "d:\AI\project\zulong_beta4\openclaw_bridge\logs\*.log" -Tail 100
```

**SearXNG Docker 日志**：
```powershell
docker logs searxng --tail 50
```

### 2. 检查工具调用

在日志中搜索以下关键词：
- `WebSearchTool` - 网络搜索工具
- `OpenClawSearchTool` - OpenClaw 搜索工具
- `OpenClawPlugin` - 网页抓取工具
- `AutoGPTPlanner` - 任务规划工具
- `OpenManusReasoner` - 推理分析工具
- `ClineCoder` - 代码生成工具

### 3. 评估输出质量

**检查点**：
- [ ] 信息是否准确
- [ ] 结构是否清晰
- [ ] 分析是否有深度
- [ ] 建议是否可执行
- [ ] 是否完成所有子任务

## 🐛 常见问题排查

### 问题 1：工具未调用

**现象**：系统直接回答，未调用任何工具

**可能原因**：
- L2 未识别到需要工具的场景
- 工具配置未正确加载
- 技能包未激活

**解决方法**：
1. 检查技能包加载日志
2. 确认工具注册成功
3. 尝试更明确的任务描述

---

### 问题 2：搜索失败

**现象**：搜索工具返回错误或无结果

**可能原因**：
- SearXNG 未运行
- OpenClaw Bridge 未启动
- 网络连接问题

**解决方法**：
```powershell
# 检查 SearXNG
docker ps | Select-String searxng

# 重启 SearXNG
docker restart searxng

# 检查 OpenClaw Bridge
# 查看进程或重新启动
```

---

### 问题 3：网页抓取失败

**现象**：无法获取网页内容

**可能原因**：
- Puppeteer 未安装
- 网页反爬虫
- 超时设置过短

**解决方法**：
1. 检查 Puppeteer 安装
2. 查看 OpenClaw 日志
3. 调整超时配置

---

## 📈 性能优化建议

### 1. 并行搜索
对于需要多来源的任务，同时发起多个搜索请求。

### 2. 缓存结果
对于相同查询，使用缓存避免重复搜索。

### 3. 智能降级
当主要工具失败时，自动切换到备用工具。

### 4. 超时控制
为每个工具调用设置合理的超时时间。

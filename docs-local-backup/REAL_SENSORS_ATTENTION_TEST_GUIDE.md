## 📊 测试场景详解

### 测试 1: 摄像头人脸检测 + 静默注意模式

**目的**: 验证静默注意模式在真实摄像头下的表现

**预期行为**:
- ✅ 摄像头持续采集 (30fps)
- ✅ 人脸出现时生成事件
- ✅ 人脸持续存在时静默 (不生成事件)
- ✅ 人脸消失时生成事件

**成功标准**:
- 检测到人脸比例 > 50%
- 静默注意效率 > 80% (大部分帧不生成事件)

**操作提示**:
- 确保光线充足
- 人脸正对摄像头
- 保持 2-3 秒不动，然后离开

---

### 测试 2: 麦克风语音检测

**目的**: 验证语音事件触发机制

**预期行为**:
- ✅ 麦克风采集音频 (16kHz)
- ✅ 检测到声音能量
- ✅ 创建语音事件

**成功标准**:
- 音频能量 > 0.01
- 事件创建成功

**操作提示**:
- 对着麦克风清晰说话
- 音量适中

---

### 测试 3: 紧急事件中断模拟

**目的**: 验证紧急事件中断流程

**场景**:
1. 用户正在与机器人对话 (普通任务)
2. 摄像头检测到用户摔倒 (紧急事件)
3. 系统中断对话，处理紧急事件
4. 紧急事件处理完毕，恢复对话

**预期行为**:
- ✅ 普通任务执行
- ✅ 紧急事件触发中断
- ✅ 任务冻结并保存快照
- ✅ Prompt 重组与注入
- ✅ 任务恢复

**成功标准**:
- 中断次数 = 1
- 冻结任务数 = 1
- 恢复任务数 = 1

---

### 测试 4: 端到端测试 (结合真实摄像头)

**目的**: 验证完整注意力机制流程

**流程**:
1. 摄像头持续监控 (静默注意)
2. 检测到人脸出现 → 生成事件 → L2 处理
3. 人脸持续存在 → 静默 (不生成事件)
4. 人脸消失 → 生成事件 → L2 处理

**预期行为**:
- ✅ 状态变化检测准确
- ✅ 事件生成符合静默注意模式
- ✅ L2 正确处理事件
- ✅ 中断恢复机制正常

**成功标准**:
- 检测到人脸状态变化
- 事件时间线合理
- L2 处理 Prompt 数匹配事件数

## 🔍 故障排查

### 问题 1: 摄像头无法初始化

**错误信息**:
```
❌ 摄像头初始化失败：Camera index out of range
```

**解决方案**:
1. 检查摄像头连接
2. 修改设备索引:
   ```python
   # 在脚本中修改
   camera = CameraDevice(device_index=0)  # 尝试 0 或 1
   ```
3. 测试摄像头是否被其他程序占用

---

### 问题 2: 未检测到人脸

**错误信息**:
```
⚠️  未检测到人脸，请检查:
   1. 摄像头是否正常工作
   2. 光线是否充足
   3. 人脸是否在摄像头范围内
```

**解决方案**:
1. 确保光线充足 (避免逆光)
2. 人脸正对摄像头，距离 30-50cm
3. 检查摄像头分辨率设置
4. 调整人脸检测阈值:
   ```python
   # 在 l1c_vision_silent_plugin.py 中
   FACE_CONFIDENCE_THRESHOLD = 0.7  # 降低阈值
   ```

---

### 问题 3: 麦克风无法录音

**错误信息**:
```
❌ 录音失败
```

**解决方案**:
1. 检查麦克风权限 (Windows 设置 → 隐私 → 麦克风)
2. 检查麦克风设备列表:
   ```python
   mic = MicrophoneDevice()
   devices = mic.list_devices()
   print(devices)
   ```
3. 指定设备索引:
   ```python
   mic = MicrophoneDevice(device_index=0)  # 尝试不同索引
   ```

---

### 问题 4: 测试卡住或无响应

**解决方案**:
1. 按 Ctrl+C 中断测试
2. 检查日志输出
3. 降低测试时长:
   ```python
   # 修改测试时长
   test_duration = 10  # 从 15 秒改为 10 秒
   ```

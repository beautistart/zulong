## 项目速览

**一句话定位**: 祖龙 (ZULONG) 是一套面向 AI 智能体和机器人的认知操作系统，通过"图记忆 + 任务编排 + 代码锚点"三角闭环架构，为 Agent 提供"理解 - 规划 - 执行 - 学习"的完整认知循环能力。

| 指标 | 数据 |
|------|------|
| **核心代码量** | 30,000+ 行，49+ 核心文件 |
| **开发模式** | 一人独立完成全部架构设计与代码实现 (OPC 模式) |
| **迭代轮次** | 26 个版本迭代 (v1.6 -> v2.6)，2 年+ 持续开发 |
| **生产级模块** | 3 个核心模块达到生产级成熟度 |
| **竞品对标** | 全球范围 8 个深度竞品调研，三角闭环架构全市场唯一 |
| **产品落地** | 记忆 MCP 插件: `pip install zulong-memory`，面向 $50B+ 编程 Agent 市场 |

## 一、项目背景与市场机遇

### 1.1 AI Agent 市场进入生产级阶段

2026 年 AI Agent 市场正从"Demo 级演示"进入"生产级部署"阶段。三个细分赛道与本项目直接相关:

| 细分赛道 | 市场规模 (2026E) | 成熟度 | 祖龙切入点 |
|---------|----------------|--------|----------|
| **Agent 记忆层** | 早期 (<$500M) | 萌芽期 | **主攻方向** -- 图记忆 MCP 服务 |
| **Agent 编排框架** | ~$5B | 成长期 | 差异化方向 -- 深度可靠性保障 |
| **具身机器人 AI** | ~$20B | 导入期 | 长线方向 -- 认知大脑后端 |

### 1.2 四大产业趋势

**趋势一: 记忆成为 Agent 核心竞争力**

Harrison Chase (LangChain 创始人, 2026): "记忆是长周期的 Context Engineering，能构筑深厚壁垒。" Anthropic 2026 趋势报告: "In 2026, agents will be able to work for days at a time." 长程 Agent 的核心依赖是持久记忆系统。

**趋势二: 端侧 AI 崛起**

4B-8B 参数模型在端侧设备上运行成为趋势，但所有主流框架假设 70B+ 云端模型，无人为小模型提供可靠运行时。这是一个被忽视的蓝海市场。

**趋势三: MCP 协议标准化**

Anthropic 主导的 MCP 协议成为 Agent 工具接入的事实标准。MCP 生态 2026 年将爆发，但"记忆"和"任务编排"层的 MCP 服务供给严重不足。

**趋势四: 具身 AI 认知层缺位**

ROS 2 解决通信、NVIDIA Isaac 解决仿真与运动控制，但"机器人如何理解、记忆、规划"的认知层，目前无标准产品。AEROS 论文 (arXiv:2604.07039, 2025.4) 提出 Single-Agent Robot Principle，学术上刚形成共识。

## 二、核心痛点与市场需求

### 2.1 三大未被满足的核心需求

| 需求 | 痛点描述 | 满足度 |
|------|---------|--------|
| **跨会话长期记忆** | 每次新会话从零开始，不记得项目架构、技术选型、踩过的坑 | 低 |
| **小模型可靠执行** | 端侧/低成本场景需要 4B-8B 模型可靠完成任务，所有框架假设大模型 | 空白 |
| **机器人认知能力** | 机器人需要理解、记忆、规划能力，不仅是运动控制 | 低 |

### 2.2 现有方案缺陷

当前 MCP 记忆市场已有早期产品 (Mem0 OpenMemory / Zep Graphiti MCP / Eve Memory 等)，但均存在共性缺陷:

| 缺陷维度 | 现有方案 | 祖龙解决方案 |
|---------|---------|------------|
| 记忆结构 | 扁平向量 + KV 存储 | **异构图 (9 种节点 + 7 种边)** |
| 检索方式 | 语义相似度搜索 (top-k 全量注入) | **BFS 扩散激活 + FAISS 双路径 (认知级过滤)** |
| 衰减/遗忘 | 无 (无限累积) 或 LLM 去重 | **艾宾浩斯曲线 + 6 级重要度分层** |
| 关联发现 | 无 / LLM 显式抽取 (高成本) | **赫布学习自动发现 (零 LLM 成本)** |
| 任务编排 | 无 | **TaskGraph 递归有向图** |
| 上下文管理 | 全量注入 (挤占推理空间) | **三模式注意力窗口 + graph_boost** |

### 2.3 目标用户群

| 用户群体 | 场景 | 优先级 |
|---------|------|--------|
| **编程开发者** | 使用 Claude Code/Cursor/Qoder 等编程 Agent，需跨会话记忆 | 最高 (快速落地) |
| **AI 应用开发者** | 构建 LLM 应用产品，需深度可靠性保障 | 高 |
| **机器人厂商/研究者** | 为机器人赋予认知能力，需认知大脑集成 | 中 (长线方向) |

## 三、产品定位与战略布局

### 3.1 双轨并行策略

| 产品线 | 定位 | 目标 | 优先级 |
|--------|------|------|--------|
| **祖龙记忆 MCP** | 编程 Agent 的跨会话记忆插件 | 快速落地、验证市场、拉取投资 | **P0** |
| **祖龙主线** | 机器人/LLM 认知操作系统 | 长期独立研发，完整 AI 系统 | P1 |

**双向反哺机制**:
- MCP 产品验证记忆架构 -> 反馈改进主线记忆引擎
- 主线的图记忆研究成果 -> 增强 MCP 产品的技术深度
- MCP 产品的收入 -> 支撑主线长期研发

### 3.2 快速落地: 记忆 MCP 服务

**产品定位**: 祖龙记忆 MCP **不是编程 Agent，而是编程 Agent 的记忆层**。

- 不跑 LLM、不做推理 -- 纯状态管理服务
- 不替代任何编程 Agent -- 增强它们
- 编程 Agent 自己的模型同时作为祖龙的 LLM -- 深度融合

**用户体验**: `pip install zulong-memory` -> `zulong-memory init` -> 配置 MCP 连接 -> 正常使用编程 Agent

**MVP 工具集 (7 个)**:

| 类别 | 工具名称 | 功能 |
|------|---------|------|
| 记忆工具 | zulong_memory_search | 图记忆语义检索 |
| 记忆工具 | zulong_memory_write | 写入新记忆节点 |
| 记忆工具 | zulong_memory_list | 列出记忆概览 |
| 任务工具 | zulong_task_view | 查看任务图状态 |
| 任务工具 | zulong_task_create | 创建任务节点 |
| 任务工具 | zulong_task_update | 更新任务状态 |
| 上下文工具 | zulong_context_inject | 智能上下文注入 |

### 3.3 长线方向: 机器人认知大脑

**核心理念**: 祖龙不做运动控制，而是作为"后端认知大脑"赋予机器人理解、记忆、规划能力，通过 L1 层桥接 ROS 2 / 厂商 SDK 等机器人原生系统。

**架构模型**:

| 层级 | 组件 | 功能 |
|:---:|------|------|
| **上层** | 祖龙 L2 认知大脑 | FC Loop + TaskGraph + MemoryGraph |
| **中层** | 祖龙 L1 调度中枢 + 感知插件 | 事件路由 + 中断仲裁 + 标准化感知接口 (IL1Module) |
| **底层** | 机器人原生系统 | ROS 2 / MoveIt2 / Nav2 / 厂商 SDK |

**学术验证**: AEROS 论文 (arXiv:2604.07039) 提出的 Single-Agent Robot Principle 与祖龙架构完全一致 -- Agent 不直接操作执行器，而是通过专用运行时层路由命令。

## 四、核心竞争优势

### 4.1 唯一的"三角闭环"架构 -- Memory x TaskGraph x Code Anchor

| 能力维度 | 祖龙 | 最接近竞品 | 差距 |
|---------|------|-----------|------|
| **记忆图谱** | 9 节点 + 7 边异构图 | Shodh-Memory 5 节点 / MemCP 4 类型 | 祖龙节点/边种类最丰富 |
| **任务编排** | TaskGraph 递归 DAG + 可视化 | LangGraph (DAG) | 祖龙与记忆图原生集成 |
| **代码锚点** | Code Anchor 双向追踪 (代码<->任务<->记忆) | 无竞品实现 | **市场唯一** |
| **三者联动** | 记忆指导任务分解 -> 任务驱动代码修改 -> 代码变更回写记忆 | 无竞品实现闭环 | **市场唯一** |

**意义**: 任何单一竞品只覆盖其中一个维度。祖龙的三角闭环使 Agent 在长程任务中具备"理解 -> 规划 -> 执行 -> 学习"的完整认知循环。

### 4.2 认知科学驱动的记忆系统

祖龙是市场上唯一将认知科学理论系统性工程化的 Agent 记忆系统:

| 特性 | 祖龙 | Mem0 | Zep | Shodh-Memory |
|------|------|------|-----|-------------|
| **图结构** | 9 节点 + 7 边异构 | 扁平 KV | 实体-关系图 | 5 节点 |
| **三维标签** | 温度/重要性/时间域 | 无 | 无 | 无 |
| **Hebbian 学习** | 渐近饱和 + 自动创建边 | 无 | 无 | 有 (简单) |
| **Ebbinghaus 衰减** | 6 级重要度半衰期 + LLM 审查 | 无 | 双时间戳失效 | 混合指数+幂律 |
| **扩散激活** | BFS + FAISS 双路并行 | 无 | 无 | spreading activation |
| **LLM 摄入成本** | 0-1 次/轮 | 1-2 次 | 6-7+ 次 | 1-2 次 |

**祖龙独有**: 三维标签体系 (温度/重要性/时间域) 在所有调研产品中无人实现，使记忆检索可同时按"活跃度""重要程度""时间范围"三个轴精准过滤。

### 4.3 极致的小模型可靠性保障

祖龙是市场上**唯一**面向 4B-8B 小模型设计完整可靠性方案的系统:

| 保障层 | 机制 | 说明 |
|-------|------|------|
| 第 1 层 | 5 层防护链 | 格式修复 -> 安全过滤 -> 幻觉检测 -> 一致性校验 -> 降级回退 |
| 第 2 层 | CB 6 信号熔断 | 连续失败/超时/资源耗尽/异常率/依赖故障/手动触发 |
| 第 3 层 | 专家热切换 (<10ms) | 7 专家类型 (通用/逻辑/创意/导航/操控/视觉/TTS)，LRU 淘汰 |
| 补充 | 双脑容器 | 左脑 temp=0.3 逻辑推理 / 右脑 temp=0.8 创意生成 |
| 补充 | 小模型兜底 | RTX 3060 6GB 即可运行，4bit 量化 |
| 补充 | 两阶段意图分类 | 粗分类 -> 细分类，减少大模型调用 |
| 补充 | 双实例中断 | 高优先级任务安全中断低优先级执行 |
| 补充 | 多级重试与降级 | 自动重试 -> 模型降级 -> 功能降级 -> 安全停止 |

**对比**: AutoGen/CrewAI/LangGraph 均假设底层 LLM 调用可靠，无此类补偿设计。

### 4.4 三层协同上下文管理

竞品框架 (LangGraph/AutoGen/CrewAI) 无上下文预算管理机制。祖龙通过三层协同实现同等上下文窗口下 **3-5 倍于竞品的有效信息密度**:

| 层级 | 机制 | 上下文空间贡献 |
|------|------|--------------|
| **图记忆层** | BFS 扩散选择性激活 + Temperature 冷热路由 | 认知级过滤，上下文占用降低 60-80% |
| **注意力窗口** | 三模式 (GLOBAL/FOCUS/SINGLE_CHAIN) + graph_boost | 预算级裁剪，淘汰低权重历史 |
| **L3 专家分流** | TTS/运动控制/深度推理独立处理 | 领域级分流，减少 30-50% 专业上下文 |

## 六、商业模式与市场空间

### 6.1 商业化路径

| 路径 | 可行性 | 市场空间 | 建议优先级 |
|------|--------|---------|----------|

### 6.2 变现模型 (记忆 MCP 方向)

| 层级 | 功能 | 定价 |
|------|------|------|
| 免费版 | 本地存储、单项目、基础记忆 | $0 |
| Pro 版 | 云同步、多项目、高级搜索、经验系统 | $9-19/月 |

### 6.3 市场机会

- 编程 Agent 市场 2026 年预计 **$50B+**
- 所有主流编程 Agent (Claude Code/Cursor/Qoder/Windsurf) 都缺少跨会话记忆 (公认痛点)

### 6.4 商业壁垒

1. **数据网络效应**: 记忆积累越多，价值越高，用户越难迁走
2. **协议先发**: 率先覆盖主流编程 Agent 的 MCP 适配
3. **技术护城河**: 三角闭环 (Memory + TaskGraph + Code Anchor) 的集成需要大量架构设计，竞品短期内难以复制

## 七、竞争窗口与风险分析

### 7.1 竞争窗口

- **差异化窗口期: 3-6 个月** -- 第三代记忆产品 (MemCP/Shodh-Memory/ClawMem) 正在追赶
- **三角闭环护城河**: Memory + TaskGraph + Code Anchor 的深度集成，当前无竞品同时实现
- **具身 AI 认知层空白**: Tesla/Boston Dynamics/宇树的运动控制成熟度越高，对认知层的需求越迫切

### 7.2 低概率被追上的核心壁垒

| 壁垒 | 理由 |
|------|------|
| 赫布学习 + 艾宾浩斯 + 三维标签 | 认知科学交叉领域深度工程，头部企业路线是"让模型更强"，不走此路线 |
| 5 层防护链 + CB + 4B 补偿 | 只有真实部署小模型才能发现这些问题，头部企业用 70B+ 不会遇到 |
| TaskGraph + MemoryGraph 深度耦合 | 双向同步 (状态->激活值、级联、地址继承) 是独有的工程积累 |
| 三层协同上下文管理 | 从检索到执行的全链路上下文管控，竞品均为单一注入模式 |

### 7.3 风险提示

| 风险 | 影响 | 应对 |
|------|------|------|
| 头部企业升级 Memory 为图结构 | 记忆优势被削弱 | 加速 MCP 产品化，建立用户基础 |
| LangGraph 原生集成 TaskGraph + 记忆 | 编排优势被削弱 | 深化三角闭环耦合，做好差异化 |
| MCP 生态竞品快速追赶 | 差异化窗口关闭 | 3-6 个月内完成 MVP 发布 |

## 八、OPC 超级个体模式

### 8.1 模式说明

本项目是 AI 时代"超级个体" (OPC, One Person Company) 模式的真实案例。创始人从设计师背景出发，独立完成了从架构设计到代码实现的全部工作，相当于传统 5-8 人技术团队的产出。

### 8.2 OPC 模式核心价值

| 维度 | 传统团队 | OPC 模式 |
|------|---------|---------|
| **决策速度** | 需协调 5-8 人 | 个人即刻决策 |
| **架构一致性** | 多人协作需对齐 | 天然一致 (一人全栈) |
| **沟通成本** | 高 (会议/文档/对齐) | 零 |
| **迭代速度** | 周级 | 日级 |
| **AI 工具利用** | 部分使用 | 深度协作 (AI 作为核心生产力工具) |

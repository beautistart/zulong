## 🌐 浏览器测试步骤

### 1. 打开对话界面

在浏览器中输入：**http://localhost:8080**

您将看到一个漂亮的对话界面，包含：
- 聊天消息区域
- 底部输入框
- 发送按钮

### 2. 基础对话测试

在输入框中输入以下内容并发送：

**测试 1：简单对话**
```
你好，请介绍一下你自己
```

**预期效果**：
- 消息显示在聊天窗口中（用户消息显示在右侧，紫色气泡）
- 祖龙回复显示在左侧（白色气泡）
- 回复应该流畅自然

**测试 2：知识问答**
```
什么是量子计算？
```

**预期效果**：
- 祖龙应该给出准确的科普解释
- 回复应该有逻辑性和条理性

**测试 3：多轮对话**
```
第一句：我想学习 Python 编程
第二句：有什么好的学习方法吗？
第三句：需要多长时间才能入门？
```

**预期效果**：
- 祖龙应该记住上下文
- 回答应该连贯，体现对之前对话的理解

## 🧠 记忆板块功能测试

### 测试 4：短期记忆测试

**步骤**：
1. 输入：`我叫小明，今年 25 岁`
2. 等待回复
3. 输入：`你还记得我叫什么吗？`

**预期效果**：
- 祖龙应该回答"你叫小明"
- 体现短期记忆功能正常

### 测试 5：长期记忆（RAG）测试

**步骤**：
1. 输入：`我喜欢吃川菜，特别是水煮鱼`
2. 等待回复
3. 输入其他话题（例如：`今天天气怎么样`）
4. 等待回复
5. 输入：`我之前说过喜欢吃什么菜？`

**预期效果**：
- 祖龙应该通过 RAG 检索到之前的对话
- 回答"你喜欢吃川菜，特别是水煮鱼"
- 体现长期记忆存储和检索功能

### 测试 6：经验学习测试

**步骤**：
1. 输入：`请帮我写一个 Python 函数，计算两个数的和`
2. 祖龙回复代码后，输入：`这个函数应该支持浮点数`
3. 祖龙修改后，输入：`很好，现在请写一个减法函数`

**预期效果**：
- 祖龙应该从第一次交互中学习
- 第二次写减法函数时应该使用类似的代码风格
- 体现经验学习和迁移能力

### 测试 7：任务拆解测试（复杂任务）

**步骤**：
```
请帮我完成以下任务：
1. 创建一个 Python 项目结构
2. 实现一个简单的 Web 服务器
3. 添加用户登录功能
```

**预期效果**：
- 祖龙应该将任务拆解为多个子任务
- 逐步完成每个子任务
- 体现任务规划和执行能力

## 🐛 常见问题排查

### 问题 1：浏览器无法连接

**症状**：打开 http://localhost:8080 显示无法访问

**解决方案**：
1. 检查 OpenClaw Bridge 是否运行（终端 5）
2. 查看是否有 "Web 适配器已启动" 日志
3. 检查端口 8080 是否被占用

### 问题 2：消息发送后无回复

**症状**：发送消息后，长时间显示"思考中..."

**解决方案**：
1. 检查祖龙主系统是否运行（终端 4）
2. 查看 L2 vLLM 服务是否正常（终端 3）
3. 检查 WebSocket 连接状态（端口 5555）

### 问题 3：记忆检索失败

**症状**：祖龙不记得之前的对话

**解决方案**：
1. 检查 config.yaml 中 `rag.enabled: true`
2. 查看日志中是否有 `[RAG]` 相关日志
3. 检查向量模型是否加载成功

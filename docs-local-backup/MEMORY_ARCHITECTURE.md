# 祖龙记忆系统架构说明

## 📚 记忆系统概述

祖龙系统的记忆系统基于 **RAG（Retrieval-Augmented Generation）** 技术，实现了类似人类的记忆机制：存储、检索、学习和应用。

### 核心理念

```
借用 -> 学习 -> 内化 -> 丢弃
```

1. **借用**：从技能包获取能力
2. **学习**：在使用过程中积累经验
3. **内化**：将经验转化为系统补丁
4. **丢弃**：学会后可以卸载技能包

---

## 🏗️ 记忆系统架构

### 三层记忆结构

```
┌─────────────────────────────────────────┐
│         长期记忆（RAG 系统）              │
│  ┌─────────────┬─────────────┬───────┐  │
│  │  经验 RAG   │  记忆 RAG   │知识RAG│  │
│  │ Experience  │  Episodic   │Knowledge│ │
│  └─────────────┴─────────────┴───────┘  │
└─────────────────────────────────────────┘
                    ↑
┌───────────────────┴───────────────────┐
│         短期记忆（对话缓存）             │
│    Short-Term Memory (STM)            │
│    - 最近 5 轮对话                      │
│    - 工作记忆                          │
└───────────────────────────────────────┘
```

### 记忆流程图

```
用户输入
    ↓
[短期记忆缓存] → 最近 5 轮对话
    ↓
[L2 推理引擎] → 生成回复
    ↓
[经验提取器] → 从对话中提取经验
    ↓
[经验 RAG] → 存储任务执行经验
    ↓
[记忆 RAG] → 存储用户个人信息/偏好
    ↓
[知识 RAG] → 存储通用知识
    ↓
[热更新引擎] → 将经验转化为系统补丁
    ↓
[下次对话] → RAG 检索增强生成
```

## 📖 三大 RAG 库详解

### 1. 经验 RAG（Experience RAG）

**功能**：存储任务执行经验和技能使用经验

**存储内容**：
- ✅ 任务成功案例
- ❌ 任务失败案例
- 🛠️ 技能使用技巧
- 📝 系统提示词归类

**标签系统**：
```yaml
importance: must_learn  # 重要经验
domain: navigation      # 领域：导航/操作/视觉/对话
```

**使用场景**：
- 用户请求类似任务时，检索历史经验
- 避免重复犯错
- 优化技能使用策略

**示例**：
```python
# 添加经验
经验："使用 web_search 工具时，先提取关键词再搜索"
标签：{"category": "skill_usage", "domain": "dialog"}

# 检索经验
用户："帮我查一下明天的天气"
→ RAG 检索到"关键词提取"经验
→ L2 先提取"明天 天气"关键词
→ 调用 web_search
```

---

### 2. 记忆 RAG（Episodic Memory RAG）

**功能**：存储用户个人信息和 episodic 记忆

**存储内容**：
- 👤 用户姓名、年龄、职业
- 🍜 用户喜好（食物、音乐、颜色等）
- 📍 用户位置信息
- 📅 重要事件和约定

**标签系统**：
```yaml
type: personal_info     # 个人信息
importance: high        # 重要程度
user_id: default        # 用户 ID
```

**使用场景**：
- 用户询问"你还记得我说过什么吗？"
- 个性化推荐和服务
- 建立长期用户关系

**示例**：
```python
# 用户说："我叫小明，喜欢吃川菜"
→ 记忆 RAG 存储：
   - 姓名：小明
   - 喜好：川菜

# 后续对话
用户："你还记得我叫什么吗？"
→ RAG 检索到"姓名：小明"
→ 回复："你叫小明呀"
```

---

### 3. 知识 RAG（Knowledge RAG）

**功能**：存储通用知识和领域知识

**存储内容**：
- 📚 百科知识
- 🔬 科学知识
- 💻 编程知识
- 📖 文档资料

**标签系统**：
```yaml
category: science       # 知识分类
source: user_dialog     # 来源
confidence: 0.9         # 置信度
```

**使用场景**：
- 知识问答
- 专业领域咨询
- 文档检索

**示例**：
```python
# 用户问："什么是量子计算？"
→ 知识 RAG 存储量子计算相关知识
→ 下次类似问题直接检索回答
→ 无需重新生成知识
```

## 🔧 记忆系统工作流程

### 记忆存储流程

```
1. 用户对话
   ↓
2. L2 生成回复
   ↓
3. ExperienceGenerator 提取经验
   - 从对话中提取关键信息
   - 生成经验文档
   ↓
4. 分类存储
   - 个人信息 → 记忆 RAG
   - 任务经验 → 经验 RAG
   - 通用知识 → 知识 RAG
   ↓
5. 向量化
   - 使用 BAAI/bge-small-zh-v1.5
   - 生成 512 维向量
   ↓
6. 存入 FAISS 向量库
   - 高效相似度搜索
   - 支持增量更新
```

### 记忆检索流程

```
1. 用户输入
   ↓
2. 向量化查询
   - 将用户问题转换为向量
   ↓
3. FAISS 相似度搜索
   - 找到最相关的 K 个记忆
   - 计算相似度分数
   ↓
4. 过滤和排序
   - 根据标签过滤
   - 按相关性和重要性排序
   ↓
5. 构建增强上下文
   - 将检索到的记忆添加到 prompt
   ↓
6. L2 生成增强回复
   - 基于记忆生成更准确的回答
```

## 🛠️ 关键技术组件

### 1. 向量模型

**模型**：`BAAI/bge-small-zh-v1.5`

**特点**：
- 🇨🇳 专门针对中文优化
- 📏 输出 512 维向量
- ⚡ 快速编码
- 💾 低显存占用

**使用方式**：
```python
from zulong.models.embedding_model import embedding_model

# 编码文本
vector = embedding_model.encode("今天天气真好")

# 编码查询
query_vector = embedding_model.encode_query("天气怎么样")
```

---

### 2. 向量存储

**后端**：`FAISS` (Facebook AI Similarity Search)

**优势**：
- 🔍 高效相似度搜索
- 📈 支持十亿级向量
- 💰 低内存占用
- 🔧 灵活索引类型

**配置**：
```yaml
vector_store_type: faiss
vector_dimension: 512
index_type: Flat  # 或 IVF、PQ 等
```

---

### 3. 经验提取器

**类**：`ExperienceGenerator`

**功能**：
- 📝 从对话中提取关键信息
- 🏷️ 自动打标签
- 📊 评估经验重要性
- 🔄 生成经验文档

**提取规则**：
```python
# 示例规则
if "我叫" in dialogue:
    extract_personal_info("name")
    
if "任务完成" in dialogue:
    extract_task_experience("success")
    
if "应该" in dialogue:
    extract_skill_experience("best_practice")
```

---

### 4. 热更新引擎

**类**：`HotUpdateEngine`

**功能**：
- 🔥 实时应用经验补丁
- 📈 持续优化系统行为
- 🎯 个性化适配
- 🔄 支持回滚

**工作流程**：
```
经验积累
    ↓
生成 SystemPatch
    ↓
应用到运行时
    ↓
观察效果
    ↓
确认有效 → 永久保存
    ↓
无效 → 回滚
```

## 📊 记忆系统配置

### config.yaml 配置

```yaml
# RAG 记忆系统配置
rag:
  enabled: true                    # 启用 RAG 系统
  memory_rag_enabled: true         # 启用记忆 RAG（长期记忆）
  experience_rag_enabled: true     # 启用经验 RAG
  knowledge_rag_enabled: true      # 启用知识 RAG
  
  # 向量模型配置
  vector_model: "BAAI/bge-small-zh-v1.5"
  vector_dimension: 512            # 模型实际输出维度
  vector_store_type: "faiss"       # 向量存储类型
  
  # 持久化配置
  persist_directory: "./data/rag"
  save_interval: 300               # 保存间隔（秒）
  
  # 检索配置
  top_k: 5                         # 默认检索数量
  min_score: 0.3                   # 最低相似度分数
```

## 📈 性能优化

### 1. 检索优化

**策略**：
- 🔍 混合搜索（向量 + 关键词）
- 🏷️ 标签过滤
- ⏰ 时间衰减
- 📊 重要性加权

### 2. 存储优化

**策略**：
- 📦 增量存储
- 🗑️ 定期清理低重要性记忆
- 📈 动态调整 top_k
- 💾 持久化到磁盘

### 3. 显存优化

**策略**：
- 🧠 L2 backup 复用 L2 core 模型
- 📉 量化向量存储
- 🔄 按需加载 RAG 库
- ⚙️ 批量处理

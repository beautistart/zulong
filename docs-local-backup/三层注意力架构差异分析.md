# 三层注意力机制架构差异分析

**分析日期**: 2026-03-26  
**目标架构**: 三层注意力机制原子任务.txt  
**当前架构**: TSD v1.7 + 现有代码

---

## 📊 架构对比总览

| 组件 | 目标架构 | 当前架构 | 差异 | 优先级 |
|------|---------|---------|------|--------|
| **注意力原子类** | `AttentionEvent`, `ContextSnapshot`, `MacroCommand` | `ZulongEvent`, `Task` | 需要新增专用原子类 | 🔴 高 |
| **L1-C 静默注意** | 独立静默插件，状态翻转时触发事件 | VisionProcessor 已实现部分功能 | 需要重构为静默模式 | 🔴 高 |
| **L1-B 注意力控制器** | `AttentionController` (冻结 + 重组) | `L1BScheduler` (简单路由) | 需要新增中断处理逻辑 | 🔴 高 |
| **L1-A 融合执行器** | `FusionController` (多模态融合) | 分散在多个执行器 | 需要统一融合逻辑 | 🟡 中 |
| **L2 快照支持** | `create_snapshot()`, `load_snapshot()` | 无 | 需要新增快照机制 | 🟡 中 |
| **共享内存** | `shared_memory['vision_target_pos']` | `shared_vision` 目录 | 需要统一数据接口 | 🟡 中 |

## 🔍 详细差异分析

### 1. 事件系统差异

#### 目标架构 (三层注意力)
```python
AttentionEvent(
    id="vis_1234567890",
    source="l1c_vision_silent",
    type=EventType.INTERACTION_TRIGGER,
    priority=5,  # 1-10
    payload={"action": "face_enter", "bbox": [...], "pos_3d": [...]},
    timestamp=1234567890.123
)
```

#### 当前架构 (TSD v1.7)
```python
ZulongEvent(
    event_id="vision_123",
    event_type=EventType.VISION_MOTION,
    priority=EventPriority.NORMAL,
    payload={"state": "MOVING", "pixels": 500},
    timestamp=1234567890.123
)
```

#### 差异点
- ✅ 当前 `ZulongEvent` 已具备基本结构
- ❌ 缺少 `AttentionLayer` 概念
- ❌ 缺少 `EventType` 的静默/触发/紧急分类
- ❌ 缺少 `source` 字段标识插件来源

**建议**: 扩展现有 `ZulongEvent` 或创建 `AttentionEvent` 作为其子类

### 2. L1-C 静默注意差异

#### 目标架构
```python
class VisionSilentPlugin(IL1Module):
    def process_cycle(self, shared_memory: dict) -> list:
        # 1. L0/L1 静默推理 (始终运行，但不报错，不生成事件)
        detections = self.model.infer(frame)
        
        # 2. 状态比较 (State Comparison) - 核心过滤逻辑
        prev_face = self.state_memory['face_detected']
        
        # 场景 A: 从无到有 (交互触发)
        if current_face and not prev_face:
            events.append(AttentionEvent(...))
            self.state_memory['face_detected'] = True
            
        # 场景 B: 紧急异常 (如摔倒检测 - 始终高优)
        elif self._detect_fall(detections):
            events.append(AttentionEvent(priority=10, ...))
            
        # 注意：如果 face 一直存在且无变化，此处不生成任何事件 (静默注意)
        
        # 将原始坐标数据写入共享内存供 L1-A 使用 (即使没有事件)
        if current_face:
            shared_memory['vision_target_pos'] = self._est_depth(current_face)
            
        return events
```

#### 当前架构
```python
class VisionProcessor(Thread):
    def feed_frame(self, frame: np.ndarray, timestamp: float):
        # 1. 存入短期记忆
        self.vstm.add_frame(frame, timestamp)

        # 2. 送入光流检测器
        result: MotionResult = self.motion_detector.process_frame(frame)
        
        # 3. 更新全局状态
        self.current_motion_state = result.state
        
        # 4. 状态翻转检测 (静默注意核心)
        if result.state != self.last_reported_state:
            self._publish_state_event(result)
            self.last_reported_state = result.state
            
            # 如果刚进入 MOVING 状态，强制记录一个高优先级事件
            if result.state == MotionState.MOVING:
                self.vstm.trigger_motion_save()
```

#### 差异点
- ✅ 当前已实现**状态翻转检测**
- ✅ 当前已实现**静默注意** (仅状态变化时发布事件)
- ❌ 当前缺少**共享内存写入** (坐标数据)
- ❌ 当前缺少**多类别检测** (人脸、手势、异常)
- ❌ 当前使用**光流法**而非**目标检测**

**建议**: 
1. 保留光流法作为底层运动检测
2. 增加目标检测插件（人脸、手势）
3. 增加共享内存写入逻辑
4. 扩展状态机（增加更多语义状态）

### 3. L1-B 注意力控制器差异

#### 目标架构
```python
class AttentionController:
    def tick(self, events: List[AttentionEvent]):
        for evt in events:
            if evt.type == EventType.EMERGENCY_ALERT or evt.is_interrupt_level():
                self._handle_interrupt(evt)
            elif self.status == "IDLE":
                self._route_to_l2_direct(evt)
            else:
                # L2 忙碌时的策略
                if evt.priority >= 7: # 高优打断
                    self._handle_interrupt(evt)
                else:
                    # 低优排队 (静默注意产生的普通交互)
                    self.event_queue.put((-evt.priority, evt))
    
    def _handle_interrupt(self, evt: AttentionEvent):
        # 1. 冻结 (Freeze)
        if self.status == "BUSY":
            self.active_snapshot = self.l2.create_snapshot()
        
        self.status = "SUSPENDED"
        
        # 2. 重组 (Recompose)
        recomposed_prompt = (
            f"[紧急事件] {evt.source} 检测到 {evt.payload}\n"
            f"[暂停的任务] {self.active_snapshot.summary}\n"
            f"请给出简短的应对指令。"
        )
        
        # 3. 注入 (Inject)
        self.l2.force_respond(recomposed_prompt, priority="IMMEDIATE")
```

#### 当前架构
```python
class L1BScheduler:
    def route_event(self, event: ZulongEvent):
        # 简单路由逻辑
        if event.event_type in [EventType.VISION_MOTION, EventType.AUDIO_WAKE_WORD]:
            self._forward_to_l2(event)
        else:
            self._handle_reflex(event)
```

#### 差异点
- ❌ 当前缺少**任务冻结机制**
- ❌ 当前缺少**上下文快照保存**
- ❌ 当前缺少**Prompt 重组逻辑**
- ❌ 当前缺少**事件队列管理**
- ❌ 当前缺少**中断优先级判断**

**建议**: 创建新的 `AttentionController` 类，或大幅重构 `L1BScheduler`

### 4. L1-A 融合执行器差异

#### 目标架构
```python
class FusionController(IL1Module):
    def process_cycle(self, shared_memory: dict) -> list:
        # 1. 读取共享内存中的多模态数据 (静默更新，无事件)
        self.sensor_fusion_buffer['vision'] = shared_memory.get('vision_target_pos')
        self.sensor_fusion_buffer['radar'] = shared_memory.get('radar_obstacles')
        self.sensor_fusion_buffer['audio'] = shared_memory.get('audio_source_pos')
        
        # 2. 读取宏观指令
        cmd = shared_memory.get('current_macro_command')
        if cmd:
            self.current_cmd = cmd
            
        # 3. 融合解算 (如果有指令)
        if self.current_cmd:
            pwm_signals = self._compute_pwm()
            self._write_motors(pwm_signals)
```

#### 当前架构
```python
class MotionExecutor:
    def execute(self, command: str):
        # 直接执行单一指令
        if command == "GRASP":
            self._control_gripper()
        elif command == "MOVE":
            self._control_motors()
```

#### 差异点
- ❌ 当前缺少**多模态数据融合**
- ❌ 当前缺少**宏观指令解析**
- ❌ 当前缺少**共享内存读取**
- ❌ 当前为**单一指令执行**而非**融合计算**

**建议**: 创建新的 `FusionController` 类，统一多模态输入

### 5. L2 快照支持差异

#### 目标架构
```python
class VLMAgent:
    def create_snapshot(self) -> ContextSnapshot:
        summary = self._ask_llm_to_summarize("请用一句话总结当前正在进行的任务。")
        
        return ContextSnapshot(
            task_id=self.current_task_id,
            summary=summary,
            full_history=self.history[-10:],
            kv_cache_ptr=self.kv_cache.clone(),
            generation_state={}
        )
        
    def load_snapshot(self, snapshot: ContextSnapshot):
        self.history = snapshot.full_history
        self.kv_cache = snapshot.kv_cache_ptr
        self.current_task_id = snapshot.task_id
```

#### 当前架构
```python
class InferenceEngine:
    async def generate(self, prompt: str):
        # 直接生成，无快照机制
        response = await self.model.generate(prompt)
        return response
```

#### 差异点
- ❌ 当前缺少**KV Cache 保存/恢复**
- ❌ 当前缺少**任务摘要生成**
- ❌ 当前缺少**历史对话管理**

**建议**: 在 `InferenceEngine` 中增加快照方法

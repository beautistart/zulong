## 🎯 测试目标

通过**引导式用户输入**，模拟真实使用场景，验证祖龙系统的核心功能：

1. ✅ **动态经验注入** - 系统从失败中学习并实时调整参数
2. ✅ **Re-evaluation** - 环境变化时重新评估任务
3. ✅ **紧急中断机制** - 检测到危险时立即中止
4. ✅ **任务进度管理** - 根据进度智能决策继续/重新规划

## 📋 测试场景详解

### 场景 1: 动态经验注入

**测试目的**: 验证系统从失败中学习的能力

**场景描述**:
```
机器人抓取杯子时，由于力度不足导致杯子滑落。
系统需要从这次失败中学习，并调整抓取力度参数。
```

**测试步骤**:
1. 输入失败描述（或直接回车使用默认）
2. 系统自动生成热补丁
3. 验证参数调整结果

**预期结果**:
```
✅ 学习成功！
   经验 ID: exp_1775147xxx
   类型：failure
   参数调整:
     - GRIP_FORCE: 0.7 (从 0.5 增加)
     - GRAB_THRESHOLD: 0.6

💡 系统成长:
   下次抓取杯子时，力度将自动调整为 0.7，避免滑落！
```

**关键验证点**:
- ✅ 经验事件驱动触发
- ✅ 补丁自动生成
- ✅ 参数实时调整
- ✅ 多层级应用（L0/L1-A/L1-B）

---

### 场景 2: Re-evaluation

**测试目的**: 验证环境变化时的重评估能力

**场景描述**:
```
机器人正在执行"把梨放到桌子上"的任务，
但环境发生了变化（梨掉到地上了）。
系统需要检测到变化并重新规划。
```

**测试步骤**:
1. 输入任务 ID（或使用默认）
2. 选择环境变化类型：
   - [1] 梨掉到地上了
   - [2] 用户移动到另一个房间
   - [3] 梨消失了
   - [4] 无变化
3. 系统重新评估并给出决策

**预期结果**:
```
📊 评估结果:
   决策：REPLAN
   原因：物体位置发生变化

💡 解释：🔄 重新规划 - 环境变化需要新策略
```

**决策规则**:
| 环境变化 | 预期决策 | 说明 |
|---------|---------|------|
| 物体移动 | REPLAN | 需要新策略 |
| 物体消失 | ABORT | 任务条件不满足 |
| 用户移动 | REPLAN | 调整跟随策略 |
| 无变化 | RESUME | 继续执行 |

---

### 场景 3: 紧急中断

**测试目的**: 验证紧急情况下的快速响应能力

**场景描述**:
```
机器人正在执行任务时，检测到紧急情况。
系统需要立即中止所有任务，优先处理紧急事件。
```

**测试步骤**:
1. 输入紧急命令（或直接回车使用默认）
2. 系统检测紧急关键词
3. 验证中断响应

**紧急关键词**:
```
["停下", "停止", "救命", "紧急", "危险"]
```

**预期结果**:
```
🚨 紧急事件触发！
   优先级：10（最高）
   动作：立即中止所有任务

✅ 紧急事件已处理
   当前任务已中止
   系统进入紧急响应模式

💡 系统响应:
   1. 立即停止所有动作
   2. 播放确认语音：'已停止，请问有什么帮助？'
   3. 等待用户进一步指令
```

**关键验证点**:
- ✅ 紧急关键词检测
- ✅ 最高优先级处理
- ✅ 任务立即中止
- ✅ 紧急响应模式

---

### 场景 4: 任务进度管理

**测试目的**: 验证基于进度的智能决策能力

**场景描述**:
```
任务被中断后，系统需要根据进度决定:
- 进度 > 80%: 继续执行（已完成大部分工作）
- 进度 < 20%: 重新规划（重新开始更划算）
```

**测试步骤**:
1. 输入任务进度（0-100）
2. 系统评估并给出决策
3. 理解决策规则

**预期结果**:
```
📊 评估结果:
   决策：RESUME
   原因：Task 85.0% complete, almost done

💡 决策规则:
   ✅ 高进度（>80%）: 继续执行 - 已完成大部分工作
```

**决策规则**:
| 进度范围 | 预期决策 | 说明 |
|---------|---------|------|
| > 80% | RESUME | 继续执行 |
| 50-80% | RESUME/REPLAN | 根据情况 |
| 20-50% | REPLAN | 重新规划 |
| < 20% | REPLAN | 重新开始 |

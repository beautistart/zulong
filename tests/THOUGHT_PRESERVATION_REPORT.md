## 📋 问题描述

### 问题 1: 思考过程未保存

**现象**: 
- 模型输出的思考过程 (`<think>...</think>` 标签内的内容) 显示后就消失了
- 对话历史中只保存最终回复，不保存推理过程
- 用户无法回顾模型的思考路径

**影响**:
- ❌ 降低透明度：用户无法理解模型的推理过程
- ❌ 难以调试：开发者无法分析模型的决策逻辑
- ❌ 丢失上下文：后续对话无法引用之前的思考内容

---

### 问题 2: 输出缺乏条理性

**现象**:
- 模型输出为纯文本，缺少 Markdown 格式
- 技术建议类内容没有清晰的层次结构
- 段落之间缺少明确的分隔

**对比**:

| 修复前 | 修复后 |
|--------|--------|
| 纯文本，无格式 | Markdown 格式化 |
| 段落混在一起 | 清晰的标题和分段 |
| 阅读性低 | 阅读性高，有条理 |

#### 代码实现

# 🔥 关键修复 1: 保存思考过程到对话历史
thought_content = None
if "<think>" in response and "</think>" in response:
    thought_parts = response.split("<think>")
    if len(thought_parts) > 1:
        # 提取思考内容 (可能有多段)
        thoughts = []
        for i in range(1, len(thought_parts)):
            thought_segment = thought_parts[i]
            if "</think>" in thought_segment:
                thought = thought_segment.split("</think>")[0].strip()
                thoughts.append(thought)
        
        if thoughts:
            thought_content = "\n\n".join(thoughts)
            logger.info(f"💭 [思考过程] 提取到 {len(thoughts)} 段思考内容")
            
            # 🔥 将思考过程保存到对话历史 (作为 system 消息)
            self.conversation_history.append({
                "role": "system",
                "content": f"[思考过程] {thought_content[:1000]}...",
                "timestamp": time.time(),
                "is_thought": True  # 标记为思考内容
            })
            logger.info(f"💾 [记忆保存] 思考过程已保存到对话历史")
```

#### 保存策略

1. **提取**: 从 `<think>...</think>` 标签中提取思考内容
2. **压缩**: 限制长度 (前 1000 字符) 避免占用过多 token
3. **标记**: 使用 `is_thought: True` 标记为思考内容
4. **角色**: 使用 `system` 角色保存，区别于用户和助手消息

### 修复 3: 结构化输出增强

**代码实现**:

```python
# 🔥 关键修复 2: 增强输出结构化 (添加 Markdown 格式)
if not cleaned_response.startswith("#") and not cleaned_response.startswith("```"):
    lines = cleaned_response.split("\n")
    
    # 检测内容类型
    has_list = any(line.strip().startswith(("•", "-", "*", "1.", "2.", "3.")) for line in lines)
    has_code = any("```" in line or "def " in line or "function " in line for line in lines)
    has_table = any("|" in line and line.count("|") >= 4 for line in lines)
    
    # 如果是技术建议类内容，添加结构化包装
    if any(keyword in cleaned_response.lower() for keyword in ["技术方案", "架构", "建议", "方案", "分析"]):
        # 添加标题
        if not cleaned_response.startswith("###"):
            cleaned_response = "### 📋 技术方案分析\n\n" + cleaned_response
        
        # 确保有清晰的段落分隔
        if "\n\n" not in cleaned_response:
            cleaned_response = cleaned_response.replace("\n", "\n\n")
        
        logger.info(f"✨ [结构化] 已为回复添加 Markdown 格式")
```

#### 检测规则

1. **列表检测**: 检测是否包含 `•`, `-`, `*`, `1.` 等列表标记
2. **代码检测**: 检测是否包含 ``` 或代码关键字
3. **表格检测**: 检测是否包含 `|` 分隔符
4. **内容类型**: 检测是否包含技术建议类关键词

#### 格式化规则

1. **添加标题**: 为技术建议类内容添加 `### 📋 技术方案分析`
2. **段落分隔**: 确保段落之间有 `\n\n` 分隔
3. **保留原有格式**: 如果已有 Markdown 格式，不重复添加

## 📞 技术要点

### 思考过程提取算法

```python
def extract_thoughts(response: str) -> List[str]:
    """提取 <think>...</think> 标签内的思考内容"""
    thoughts = []
    
    if "<think>" not in response or "</think>" not in response:
        return thoughts
    
    thought_parts = response.split("<think>")
    
    for i in range(1, len(thought_parts)):
        thought_segment = thought_parts[i]
        
        if "</think>" in thought_segment:
            thought = thought_segment.split("</think>")[0].strip()
            thoughts.append(thought)
    
    return thoughts
```
